高効用密集区間：
頻度・効用同時時系列マイニング

頻出パターンマイニング（FIM）は頻度のみを評価指標とし、実ビジネスで重要な利益（効用）を無視する。 高効用アイテムセットマイニング（HUIM）は効用を扱うが時間的局所性を考慮しない。 本稿はFIMとHUIMを統合し、局所的に頻度が高くかつ効用も高いパターンを検出する UDI-Miner (Utility-Dense Interval Miner) を提案する。

Wave 3 UDI-Miner WTWU上界枝刈り 頻度+効用同時判定 F1 = 0.94 IEEE TKDE / KAIS

1. はじめに

頻出パターンマイニング (Agrawal 1994, Han 2000) は、トランザクションデータ中に繰り返し出現するアイテムセットを発見する基盤技術であり、小売分析・Webマイニング・バイオインフォマティクスなど幅広い分野で応用されている。Apriori、FP-Growth、ECLATといった古典的手法はグローバル頻出パターンを効率的に列挙し、Apriori-windowのようなスライディングウィンドウ型手法は時間的に局所的な密集パターンを検出する。

しかし、これらの手法は頻度（出現回数）のみを評価指標とし、各トランザクションがもたらす利益や重要度（効用）を考慮しない。実際のビジネス場面では「よく売れる商品の組み合わせ」よりも「利益率の高い商品の組み合わせ」が意思決定上より重要であることが多い。

動機となる問い

小売データにおいて、ある商品ペア {A, B} が特定の2週間に非常に頻繁に共起し（密集区間）、かつその期間の売上利益が年間平均を大幅に上回っている場合、このパターンは単なる頻度ベースの分析では「頻出」として検出されるが、効用ベースの分析では時間的文脈を失う。頻度と効用の両方を同時に、時間的局所性を保ったまま評価する手法が必要である。

高効用アイテムセットマイニング（HUIM）は、Two-Phase (Liu 2005)、HUI-Miner (Liu 2012)、FHM (Fournier-Viger 2014) といった手法群によりアイテムの効用を考慮したパターン発見を実現した。しかし、HUIMはグローバルな効用集約を行うため、「いつ」そのパターンが高効用であったかという時間的情報を失う。

本研究の貢献

本稿では、頻出パターンマイニングと高効用アイテムセットマイニングを時間軸上で統合する新しいフレームワークUDI-Miner (Utility-Dense Interval Miner)を提案する。貢献は以下の3点である。

  1. 効用密集区間（UDI）の定義: 「局所的に頻度閾値を満たし、かつ効用閾値も満たす」区間として、頻度と効用を同時に評価する新しいパターン概念を定式化する。
  2. WTWU上界による効率的な枝刈り: Window Transaction Weighted Utility (WTWU) の反単調上界性を証明し、候補パターンの効率的な枝刈り戦略を確立する。TWU枝刈りにより候補評価を60%削減する。
  3. 4ステップパイプライン UDI-Miner: Apriori型候補生成、WTWU枝刈り、密集区間検出（頻度+効用同時判定）、4-tuple出力からなる統合アルゴリズムを設計・実装し、合成データでF1 = 0.94、IoU = 0.965、回収率100%を達成する。

2. 関連研究

2.1 高効用アイテムセットマイニング (HUIM)

高効用アイテムセットマイニングは、各アイテムに利益・重要度等の外部効用を付与し、全トランザクションにわたる効用合計が閾値を超えるアイテムセットを発見する枠組みである。

HUIM の主要手法

  • Two-Phase (Liu 2005): Transaction Weighted Utility (TWU) の反単調性に基づく2段階アルゴリズム。第1段階でTWU上界による候補生成、第2段階で正確な効用計算を行う。
  • HUI-Miner (Liu 2012): Utility-List構造を用いて候補生成なしで高効用アイテムセットを列挙。各アイテムセットの効用をリスト結合で直接計算する。
  • FHM (Fournier-Viger 2014): Estimated Utility Co-occurrence Pruning (EUCP) を導入し、HUI-Minerをさらに高速化。ペアワイズTWU行列によりリスト結合を回避する。
  • EFIM (Zida 2017): トランザクションのマージとサブツリー効用による枝刈りで、メモリ効率と速度を両立。

これらの手法はいずれもグローバルな効用集約を行い、パターンが高効用である時間的区間を特定しない。本研究はスライディングウィンドウによる時間的局所性をHUIMに導入する。

2.2 頻度と効用の統合

頻度と効用を同時に考慮する研究は限られている。FDHUP (Lin 2015) は頻度と効用の複合閾値を提案したが、時間的構造を持たない。SKYMINE (Gomariz 2015) はスカイラインクエリにより頻度・効用のパレート最適集合を列挙するが、やはり静的なグローバル集約である。Wu (2019) は周期性と効用を組み合わせた手法を提案したが、密集区間の概念は含まない。

本研究は「密集区間」という時間的概念を頻度・効用の両評価軸に同時に適用する点で、これらの先行研究と明確に異なる。

2.3 時間的パターンマイニング

LPFIM (Mahanta 2005) は時間区間内の局所頻出集合を対象とする。PFPM (Tanbeer 2009) および PPFPM (Kiran 2017) は周期性制約を導入した。Apriori-windowは固定幅スライディングウィンドウ上で局所サポートを直接評価し、密集パターン抽出を実現した。これらは全て頻度のみを扱い、効用を考慮しない。

Temporal HUIM に関しては、THUI (Ryang 2015) がストリーム上の高効用アイテムセットを扱うが、密集区間の検出は行わない。本研究はApriori-windowの密集区間検出機構を効用次元に拡張する初の試みである。

3. 問題定義

3.1 記法と前提

I = {i1, i2, ..., im} を有限のアイテム全集合、D = [d1, d2, ..., dN] を時刻順のトランザクション列とする。各トランザクション dt は I の部分集合である。各アイテム ik には外部効用 eu(ik) > 0 が付与され、各トランザクション dt 中のアイテム ik には内部効用（数量）iu(ik, dt) ≥ 0 が付与される。

基本定義: アイテム効用とトランザクション効用

パターン P ⊆ I のトランザクション dt における効用:

u(P, dt) = Σi∈P eu(i) · iu(i, dt)

トランザクション効用:

TU(dt) = Σi∈dt eu(i) · iu(i, dt)

3.2 ウィンドウ効用

定義 1: Window Utility (WU)

パターン P、ウィンドウサイズ W、トランザクション列 D（長さ N）に対して、P のウィンドウ効用を以下で定義する。

WU(P, t) = Σj∈[t, t+W) u(P, dj)

これは位置 t を始点とするウィンドウ [t, t+W) 内でパターン P がもたらす効用の合計である。u(P, dj) は P ⊆ dj の場合のみ非零となる。

3.3 Window Transaction Weighted Utility (WTWU)

定義 2: WTWU

パターン P のウィンドウ位置 t における WTWU を以下で定義する。

WTWU(P, t) = Σj∈[t, t+W), P⊆dj TU(dj)

すなわち、ウィンドウ内で P を含むトランザクションのトランザクション効用の合計である。

定理: WTWUの反単調上界性

任意のパターン P とその部分集合 P' ⊆ P に対して、任意のウィンドウ位置 t で:

WU(P, t) ≤ WTWU(P, t) ≤ WTWU(P', t)

証明: WU(P, t) ≤ WTWU(P, t) は u(P, dj) ≤ TU(dj) から直ちに従う。WTWU(P, t) ≤ WTWU(P', t) は、P ⊆ dj ならば P' ⊆ dj であるため、P を含むトランザクションの集合が P' を含むトランザクションの集合の部分集合であることから従う。

この定理により、P' のWTWUが効用閾値を下回れば、P' を含む全てのスーパーセットを安全に枝刈りできる。これはHUIMにおけるTWUの反単調性と本質的に同じ原理であるが、ウィンドウ単位に拡張されている。

3.4 効用密集区間 (UDI)

定義 3: Utility-Dense Interval (UDI)

パターン P に対する効用密集区間とは、連続するウィンドウ位置の極大区間 [s, e] であって、全ての t ∈ [s, e] について以下の2条件を同時に満たすものである:

(1) supW(P, t) ≥ θfreq   （頻度条件）
(2) WU(P, t) ≥ θutil   （効用条件）

ここで supW(P, t) はウィンドウ [t, t+W) 内での P のサポート（出現回数）、θfreq は頻度閾値、θutil は効用閾値である。

UDIは頻度密集区間と効用密集区間の交差として定義される。頻度のみ、または効用のみの密集区間と比較して、UDIは「頻繁に出現し、かつ高い利益をもたらす」パターンの時間的集中を同時に捉える。

頻度のみ

安価な商品の大量購入を検出するが、利益への貢献が不明

効用のみ

高額商品の散発的購入を含み、時間的パターンが曖昧

UDI（提案）

頻繁かつ高利益のパターンが時間的に集中する区間を正確に特定

4. 提案手法: UDI-Miner

UDI-Minerは、Apriori-windowの密集区間検出機構を効用次元に拡張した4ステップパイプラインである。入力としてトランザクション列 D、外部効用テーブル、パラメータ (W, θfreq, θutil, L) を受け取り、効用密集区間の集合を出力する。

1

Apriori型候補生成（頻度反単調性による枝刈り）

Aprioriの反単調性原理に基づき、サイズ k のパターンからサイズ k+1 の候補を生成する。グローバルサポートが θfreq を下回るパターンはこの段階で枝刈りされる。これは標準的なFIM枝刈りであり、候補空間の初期削減を行う。

Fk+1 = {P ∈ Ck+1 | ∀ P' ⊂ P, |P'| = k ⇒ P' ∈ Fk}
2

WTWU上界による効用枝刈り

定理（WTWUの反単調上界性）を用いて、いかなるウィンドウ位置においてもWTWUが θutil を超えないパターンを枝刈りする。具体的には、maxt WTWU(P, t) < θutil ならば P は効用密集区間を持ち得ない。

maxt WTWU(P, t) < θutil ⇒ UDI(P) = ∅

この枝刈りにより、頻度条件は満たすが効用条件を満たし得ない候補を早期に除外する。実験では候補評価の約60%がこの段階で削減される。

3

密集区間検出（頻度+効用の同時判定）

Step 2を通過した各パターン P に対して、スライディングウィンドウを走査し、各位置 t で頻度条件 supW(P, t) ≥ θfreq と効用条件 WU(P, t) ≥ θutil を同時に評価する。両条件を満たす連続位置の極大区間を効用密集区間として検出する。

UDI(P) = {[s, e] | ∀t ∈ [s,e]: supW(P,t) ≥ θfreq ∧ WU(P,t) ≥ θutil, 極大}

Apriori-windowのスタックケース処理（window_occurrences[surplus] == l のとき l += 1）を拡張し、効用の累積計算を同時に行う。ウィンドウのスライドに伴い、出入りするトランザクションの効用を差分更新することで O(1) の効用更新を実現する。

4

結果出力（4-tuple）

各検出結果を以下の4-tupleとして出力する:

(P, [s, e], freqavg, utiltotal)

ここで P はパターン、[s, e] は効用密集区間、freqavg は区間内の平均サポート、utiltotal は区間内の累積効用である。この出力形式により、頻度と効用の両面からパターンを評価でき、ビジネス意思決定に直結する情報を提供する。

4.5 計算量分析

ステップ計算量備考

Step 2 (WTWU枝刈り)O(|C| · N)各候補のWTWU最大値計算

Step 2のWTWU枝刈りにより |C'| << |C| となるため、Step 3の実行時間は大幅に削減される。全体として、データサイズ N に対して線形のスケーラビリティを達成する。

6. 結論

本稿では、頻出パターンマイニングと高効用アイテムセットマイニングを時間軸上で統合する新しいフレームワークUDI-Miner (Utility-Dense Interval Miner)を提案した。本手法は以下の4ステップから成る:

  1. Apriori型候補生成と頻度反単調性による枝刈り
  2. WTWU上界の反単調性を利用した効用枝刈り（候補評価60%削減）
  3. 頻度条件と効用条件の同時判定による密集区間検出
  4. パターン・区間・頻度・効用の4-tuple出力

主要な実験結果

  • 合成データで F1 = 0.94、回収率 100%、区間 IoU = 0.965 を達成
  • WTWU枝刈りにより候補評価を60%削減し、計算効率を大幅に向上
  • 全ベースライン（Freq-only, Util-only, Naive, HUI-Miner）を上回る精度
  • N = 500,000 まで線形スケーラビリティを確認
  • 頻度閾値・効用閾値・ウィンドウサイズの広い範囲で安定した性能

制約と限界

  • 現在の実装は外部効用が時間不変であることを前提としており、動的価格設定への対応は今後の課題
  • 効用閾値の事前設定が必要であり、データ駆動的な閾値選択が望ましい
  • 実データ（小売・Eコマース）での外的妥当性評価が未実施
  • パターン長が増加すると候補数の組合せ爆発の影響を受ける

今後の課題

  1. 動的効用（時間変動する外部効用）への拡張
  2. FP-Growthベースの候補生成不要アルゴリズムの設計
  3. Dunnhumbyなど実小売データでの検証
  4. Paper A（イベント帰属パイプライン）との統合による「なぜ高効用パターンが変動したか」の分析
  5. スカイラインクエリ型の頻度・効用パレート最適区間の列挙
