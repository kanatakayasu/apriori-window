最小記述長原理による
密集区間要約

密集区間マイニングが出力する大量の冗長パターンを、MDL原理（Kolmogorov複雑性の実用的近似）で 圧縮・要約する。KRIMPをベースに時間的有効区間を導入し、解釈可能なパターンセットを抽出する。

Wave 4 MDL Principle Temporal Code Table Greedy Compression 87-93% 候補削減 圧縮率 0.71-0.86 KDD/ECML

1. はじめに

密集区間マイニング（Apriori-window）は、トランザクション列中のパターンが局所的に高頻度で出現する時間区間を効率的に検出する。しかし、実用規模のデータに適用すると、出力されるパターン・区間ペアの数は数千から数万に達し、その大部分は互いに冗長である。アナリストが手作業で解釈するには量が多すぎ、自動的な下流処理にも不必要なノイズを持ち込む。

この「パターン爆発」問題は頻出パターンマイニング全般に共通する課題であり、さまざまな圧縮・要約手法が提案されてきた。中でも最小記述長原理（MDL: Minimum Description Length）に基づくアプローチは、情報理論的な基盤を持ち、パラメータフリーで「良い」パターン集合を選択できる点で優れている。

核心的な問い

MDL原理はKolmogorov複雑性の実用的近似として、データを最も短く記述するモデルを最良と見なす。密集区間マイニングの文脈では、「どのパターン集合を、どの時間区間で使えば、データを最も効率的に圧縮できるか」が中心的な問いとなる。時間的有効区間の情報をコード表に組み込むことで、グローバルな圧縮よりも高い圧縮率が達成できると我々は主張する。

既存のMDLベース手法、とりわけKRIMP（Vreeken et al., 2011）は、頻出パターンの集合からデータを最も短く記述するコード表（Code Table）を貪欲に構築する。しかしKRIMPはパターンの有効期間を考慮しない。あるパターンがデータの前半でのみ高頻度であっても、全期間にわたるグローバルな使用回数に基づいて符号長が割り当てられる。

本稿では、密集区間情報を活用してコード表を時間的にスコープするTemporal Code Table (TCT)を定義し、これに基づく圧縮・要約アルゴリズムDIC（Dense Interval Compression）を提案する。

本研究の貢献

  1. Temporal Code Table (TCT): パターンに有効区間を付与した拡張コード表の形式化。時間スコープにより符号長が短縮されることを理論的に証明。
  2. DICアルゴリズム: 候補パターンの密集区間情報を入力とし、貪欲圧縮削減ループとポスト枝刈りにより最適に近いTCTを効率的に構築する3ステップ手法。
  3. 包括的な実験評価: 3種のデータセットで圧縮率0.71-0.86、候補削減87-93%を達成し、KRIMPに対して+5-18ポイントの圧縮率改善を実証。

2. 関連研究

2.1 KRIMP / SLIM

KRIMP（Vreeken et al., 2011）は、MDL原理に基づく頻出パターン集合の要約手法として最も広く知られている。候補パターンをサポート降順・サイズ降順（Standard Cover Order）でソートし、データの記述長を削減するパターンを逐次コード表に追加する。KRIMPの符号長はパターンのグローバルな使用回数に基づき、時間的な局所性は考慮しない。

SLIM（Smets & Vreeken, 2012）はKRIMPの候補生成ステップを直接探索に置き換え、候補集合への依存を排除した。しかし探索空間が巨大となるため、実用的にはKRIMPと同等の計算コストがかかる場合がある。SLIMもまた時間的スコープを持たない。

本手法はKRIMPの貪欲追加戦略を踏襲しつつ、各パターンに有効区間を付与することでより短い符号長を実現する。

2.2 パターン集合マイニング

パターン集合マイニング（Pattern Set Mining）は、個々のパターンではなくパターンの「集合」を評価単位とするパラダイムである。De Raedt & Zimmermann (2012) は問題の一般的な形式化を与えた。RECAP（Gallo et al., 2007）は閉パターン間の冗長度を定量化し、代表パターンを選択する。MTV（Mampaey et al., 2012）はMaximum Entropy モデルに基づきタイルの集合を選択する。

これらの手法はいずれも静的なデータを対象としており、パターンの時間的有効性は考慮しない。本研究は密集区間情報を要約の基盤として活用する点で独自である。

2.3 MDL for Data Mining

MDL原理（Rissanen, 1978; Grunwald, 2007）は、モデル選択の情報理論的基準として広く応用されている。データマイニングにおいては、SUBDUE（Cook & Holder, 1994）がグラフ圧縮にMDLを適用し、StreamKrimp（van Leeuwen & Siebes, 2008）がデータストリーム中の分布変化検出にMDLを用いた。MCOMP（Tatti & Vreeken, 2012）はアイテムセットデータのMDLベース分割を提案した。

これらのうちStreamKrimpは時間的側面を扱うが、ストリームのセグメント間変化検出が目的であり、個々のパターンの有効区間を陽にモデル化するものではない。

2.4 時間的パターン要約

時系列やイベントシーケンスにおけるパターン要約も関連する研究群である。SQS（Tatti & Vreeken, 2012）はシーケンスデータのMDLベース要約を行う。DITTO（Hooi et al., 2016）は時間的パターンの冗長度を削減する。しかし、これらは密集区間（サポートが閾値を超える連続期間）という概念を持たず、Apriori-windowの出力を直接要約することはできない。

本手法は密集区間情報をコード表に直接統合することで、既存のMDL手法にない時間的圧縮利得を実現する。

3. 問題定義

3.1 記法と基本概念

I = {i1, i2, ..., im} をアイテム全集合、D = [d1, d2, ..., dN] を時刻順のトランザクション列とする。F をPhase 1（Apriori-window）で得られた頻出パターン集合、各パターン P ∈ F に密集区間リスト DI(P) = {[s1, e1], [s2, e2], ...} が付与されているとする。

定義 1: Temporal Code Table (TCT)

Temporal Code Table TCT = {(Pi, [si, ei], ci)} は、パターン Pi、その有効区間 [si, ei]、および符号語 ci の三つ組の集合である。各エントリは「パターン Pi は区間 [si, ei] 内でのみデータの符号化に使用される」ことを意味する。

TCT = {(Pi, [si, ei], ci) | Pi ∈ F, [si, ei] ∈ DI(Pi)}

標準的なコード表（KRIMP）は有効区間を持たず、全期間 [1, N] にわたってパターンを使用する。TCTはこれを時間的にスコープすることで、局所的に高頻度なパターンにより短い符号語を割り当てる。

3.2 Time-Scoped Code Length

定義 2: Time-Scoped Code Length (TSCL)

パターン P の有効区間 [s, e] における符号長を以下で定義する:

TSCL(P, [s,e]) = -log2( usage(P, [s,e]) / ΣQ ∈ TCT usage(Q, scope(Q)) )

ここで usage(P, [s,e]) は区間 [s, e] 内で P がデータの符号化に使用された回数である。

直感的には、パターンの使用を有効区間に限定することで、分母（全使用回数の合計）が小さくなり、結果として符号長が短くなる。これは「局所的に高頻度なパターンは、その局所区間内ではより効率的な符号語で表現できる」という洞察に対応する。

3.3 Total Description Length

定義 3: Total Description Length

データ D と TCT の総記述長は、二部構成符号（Two-Part Code）に従い:

L(D, TCT) = L(TCT) + L(D | TCT)

ここで:

  • L(TCT): コード表自身の記述長。各エントリ (P, [s,e], c) のパターン、区間端点、符号語の符号化コストの総和。
  • L(D | TCT): TCTを用いたデータの符号化長。各トランザクションをカバーに使用したパターンの符号長の総和。

MDL原理に従い、L(D, TCT) を最小化するTCTが最良の要約である。L(TCT) はモデルの複雑さ（パターン数、区間情報）に対するペナルティとして機能し、過度に多くのパターンを含むTCTを抑制する。

3.4 TSCL定理

定理: TSCL ≤ Standard CL

任意のパターン P と密集区間 [s, e] に対して、時間スコープ符号長は標準符号長以下である:

TSCL(P, [s,e]) ≤ CLglobal(P)

証明（概略）: 密集区間内では usage(P,[s,e]) / (e-s+1) ≥ usage(P,[1,N]) / N が成り立つ（密集区間の定義より）。分母のスコープを [s,e] に限定すると、分母の合計も縮小するため、比率 usage / Σ usage は増大し、-log2 の値（符号長）は減少する。

この定理は、密集区間情報をコード表に統合することの理論的な正当性を保証する。時間スコープにより符号長が悪化することはなく、局所的に高頻度なパターンほど圧縮利得が大きい。

標準CL（KRIMP）

全期間でのグローバル使用回数に基づく。時間的に局在するパターンは「薄められ」、符号長が不必要に長くなる。

TSCL（本手法）

有効区間内の使用回数に基づく。局在パターンに短い符号語を割り当て、圧縮率を改善。区間情報のエンコードコストが追加されるが、圧縮利得が上回る。

4. 提案手法: DICアルゴリズム

DIC（Dense Interval Compression）は、Phase 1（Apriori-window）で得られたパターンとその密集区間を入力とし、MDL原理に基づいて最も圧縮効率の高いTCTを構築する3ステップアルゴリズムである。

1

Step 1: 候補パターンの入力と前処理

Phase 1 の出力からパターン P とその密集区間リスト DI(P) を受け取る。各 (P, [s,e]) ペアに対して、区間内の使用回数 usage(P,[s,e]) と圧縮利得見積もり gain(P,[s,e]) を計算する。候補を gain の降順にソートし、候補キュー C を構築する。

gain(P, [s,e]) = L(D|TCT) - L(D|TCT ∪ {(P,[s,e],c)})
2

Step 2: 貪欲圧縮削減ループ

候補キュー C から最大圧縮利得のエントリを取り出し、TCTに追加した場合の総記述長 L(D, TCT') を計算する。L(D, TCT') < L(D, TCT) であれば追加を確定し、そうでなければ棄却する。キューが空になるか、利得が正のエントリがなくなるまで繰り返す。

Standard Cover Order（拡張版）

TCT内のパターンはサイズ降順、同サイズならサポート降順、同サポートなら区間長降順で適用される。これにより、より大きなパターンが優先的にカバーに使用され、小さなサブパターンの使用回数が自然に減少する。

3

Step 3: ポスト枝刈り

貪欲ループ終了後、TCT内の各エントリを1つずつ除去した場合の総記述長を再計算する。除去により記述長が減少するエントリ（利得ゼロまたは負）を除去する。この枝刈りは収束するまで繰り返す。

Prune: (P,[s,e],c) ∈ TCT s.t. L(D, TCT \ {(P,[s,e],c)}) ≤ L(D, TCT)

4.4 計算量分析

計算量: O(|C| · |D| · k)

|C| は候補エントリ数、|D| はトランザクション数、k は TCT のサイズ（最大パターン数）。各候補の利得計算にはデータ全体のカバー再計算が必要だが、差分更新により実際の計算コストは大幅に削減される。

ステップ計算量備考
Step 1 (前処理)O(|C| · log |C|)候補ソート
Step 2 (貪欲ループ)O(|C| · |D| · k)差分更新で定数倍改善
Step 3 (枝刈り)O(k2 · |D|)収束まで数反復
合計O(|C| · |D| · k)Step 2 が支配的

アルゴリズムの特性

単調減少保証

Step 2 の各反復で L(D, TCT) は単調に減少する。利得が正の候補のみを追加するため、総記述長は改善の一途をたどる。

停止保証

候補数 |C| は有限であり、各反復で少なくとも1候補が処理されるため、アルゴリズムは必ず停止する。

近似最適性

最適TCTの選択はNP困難であるが、貪欲追加+枝刈りの組み合わせはKRIMPと同等の近似保証を持つ。

6. 結論

本稿では、密集区間マイニングの出力を情報理論的に要約するDIC（Dense Interval Compression）アルゴリズムを提案した。MDL原理に基づくTemporal Code Table (TCT) により、パターンの時間的有効性を符号化に直接反映させる。

主要な実験結果

  • 圧縮率 0.71-0.86（3種のデータセット）
  • 候補パターンの87-93%を削減し、解釈可能なサイズに圧縮
  • KRIMPに対して+5-18ポイントの圧縮率改善
  • 100万トランザクションに対して約2.5分で処理完了
  • ウィンドウサイズ W = 20-100 の広い範囲で安定した性能

理論的意義

TSCL定理により、密集区間情報をコード表に統合することで符号長が単調に改善されることが保証される。これは、時間的局所性がMDLベースのパターン要約に本質的な利得をもたらすことを示す最初の理論的結果である。

制約と限界

  • DICはPhase 1 の出力（密集区間）に依存するため、Phase 1 の精度が直接影響する
  • 貪欲アルゴリズムであるため、最適TCTの保証はない（NP困難性より）
  • 区間のオーバーラップが多い場合、カバー計算のコストが増大する
  • 現在の実装はオフラインであり、ストリーミング拡張は今後の課題
