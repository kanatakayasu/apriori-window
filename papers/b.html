1. はじめに

頻出パターンマイニング（FIM）は、トランザクションデータベース中で高頻度に出現するアイテムの組合せを発見する基盤的技術である。Apriori (Agrawal 1994)、FP-Growth (Han 2000)、ECLAT (Zaki 2000) に代表される古典的手法は、反単調性（anti-monotonicity）に基づく効率的な枝刈りにより、大規模データからのパターン列挙を実現してきた。

しかし、反単調性への依存はグローバルサポートが最小閾値を下回るパターンを一律に棄却する。この設計は、「通常は稀だが特定期間に局所的に密集する」パターンを原理的に発見できないという根本的限界を持つ。

動機付け: 見逃されるパターン

実応用では、グローバルに稀少なパターンこそが最も情報量の高いシグナルであることが多い。例えば：

  • 季節商品: 水着やスキー用品は年間を通じては低サポートだが、特定シーズンに集中購買される
  • 限定キャンペーン品: プロモーション期間にのみ共起する商品ペアは、通年では稀少に見える
  • 新興トレンド: 新商品や新カテゴリは導入初期に局所的なバーストを示すが、全体のサポートは低い
  • サイバー攻撃シグネチャ: 攻撃パターンは通常は稀少だが、攻撃期間中に集中して出現する

これらのパターンは従来の FIM では閾値以下として棄却され、稀少パターンマイニングでは時間的局所性を考慮しないため、「いつ」密集するかを特定できない。

本稿では、グローバルサポートが低い（稀少）にもかかわらず、特定の時間区間内で局所的にサポートが高い（密集）パターンを 稀少密集パターン（Rare Dense Pattern, RDP） として形式的に定義する。さらに、Apriori-window のスライディングウィンドウ機構をベースとした効率的な3ステップアルゴリズム RDP-Miner を提案し、RDP の発見を実現する。

本研究の貢献

  1. 新パターンクラス RDP の定義: グローバル稀少性と局所密集性を同時に要求する新しいパターンクラスを形式的に定義。従来の FIM でも稀少パターンマイニングでも発見できなかったパターン空間を明確化する。
  2. 弱反単調性定理: RDP の稀少性条件が上方閉（upward-closed）であることを証明し、候補生成における効率的な枝刈り戦略を正当化する。
  3. RDP-Miner アルゴリズム: グローバル稀少候補抽出、Apriori-window による局所密集区間検出、RDP フィルタリングの3ステップからなる統合パイプラインを設計・実装する。
  4. 包括的な実験評価: 合成データでの完全回復（Recovery 100%）、実データでの99%出力削減、線形スケーラビリティを実証する。

2. 関連研究

2.1 頻出パターンマイニング

頻出パターンマイニングの研究は30年以上の歴史を持つ。Apriori (Agrawal & Srikant, 1994) は反単調性に基づく候補生成・枝刈りフレームワークを確立し、FIM の基盤を築いた。FP-Growth (Han et al., 2000) は FP-tree による候補生成不要なアプローチで大幅な高速化を実現した。ECLAT (Zaki, 2000) は縦型（tidset）表現と集合積による効率的なサポート計算を提案した。LCM (Uno et al., 2004) は頻出・閉・極大アイテムセットの出力線形時間列挙を達成した。

これらの手法はいずれもグローバルサポート閾値を前提とし、閾値未満のパターンは候補段階で枝刈りされる。そのため、グローバルに稀少だが局所的に密集するパターン（RDP）は原理的に発見できない。

2.2 稀少パターンマイニング

稀少（低頻度）パターンの発見を目的とする研究群が存在する。RP-tree (Tsang et al., 2011) は FP-tree を稀少パターン用に拡張し、低サポートアイテムセットの効率的な列挙を実現した。Minimum support gap (Kiran & Reddy, 2011) 戦略は、アイテムごとに異なる最小サポートを設定し、稀少アイテムを含むパターンの発見を促進する。ARIMA (Szathmary et al., 2007) は最小希少（minimal rare）アイテムセットの概念を導入し、稀少パターンの簡潔な表現を提案した。

これらの手法はグローバルサポートが低いパターンを発見する能力を持つが、時間的局所性を考慮しない。すなわち、「いつ」そのパターンが密集するかを特定する機能を持たず、RDP の密集区間を検出できない。

2.3 局所的パターンマイニング

時間的な局所性を扱うパターンマイニング手法は複数提案されている。LPFIM (Mahanta et al., 2005) は時間区間内の局所頻出集合を発見するが、gap ベースの閾値に依存する。PFPM (Tanbeer et al., 2009) と PPFPM (Kiran et al., 2017) は周期性制約を導入した局所パターンマイニングを提案した。LPPM (Fournier-Viger et al., 2021) は局所性と周期性を統合的に扱う。

Apriori-window は固定幅スライディングウィンドウ上で局所サポートを直接評価し、gap パラメータに依存しない密集区間検出を実現した。本手法はこの Apriori-window を基盤として活用するが、既存の局所的パターンマイニング手法はグローバルに頻出なパターンを出発点とするため、グローバルに稀少なパターンの局所密集を発見するという RDP の問題設定には対応しない。

2.4 コントラスト・エマージングパターン

エマージングパターン (Dong & Li, 1999) はデータセット間のサポート比を捉え、コントラストセットマイニング (Bay & Pazzani, 2001) はグループ間の判別的特徴を発見する。これらはデータを予め時間区間で分割する必要があり、密集区間が未知の場合には適用できない。RDP-Miner は密集区間の検出自体をアルゴリズムに内包しており、この制約を克服する。

3. 問題定義

3.1 記法とグローバルサポート

I = {i1, i2, ..., im} を有限のアイテム全集合、D = [d1, d2, ..., dN] を時刻順のトランザクション列とする。各トランザクション dt は I の部分集合である。

定義 1: グローバルサポート

パターン P ⊆ I のグローバルサポートを以下で定義する。

supG(P) = |{t : P ⊆ dt, 1 ≤ t ≤ N}| / N

全トランザクション中で P を含むトランザクションの割合。従来の FIM はこの値が閾値以上のパターンのみを出力する。

3.2 局所サポートと密集区間

定義 2: 局所サポート（ウィンドウ内サポート）

パターン P、ウィンドウサイズ W、開始位置 t に対して、局所サポートを以下で定義する。

supW(P, t) = |{dj ∈ D | t ≤ j < t + W, P ⊆ dj}|

位置 t を始点とするウィンドウ [t, t+W) 内で P を含むトランザクション数（カウント値）。

定義 2a: 密集区間

パターン P に対する密集区間 [s, e] は、全ての t ∈ [s, e] で supW(P, t) ≥ θdense を満たす極大連続区間である。すなわち、ウィンドウ左端を s から e まで移動させた全ての位置で局所サポートが密集閾値を満たす。

3.3 稀少密集パターン（RDP）

定義 3: 稀少密集パターン（Rare Dense Pattern）

パターン P は以下の2条件を同時に満たすとき、稀少密集パターン（RDP）と呼ぶ。

(1) グローバル稀少: supG(P) < θrare
(2) 局所密集: ∃[s, e] s.t. ∀t ∈ [s, e], supW(P, t) ≥ θdense

条件 (1) はグローバルサポートが稀少閾値未満であること、条件 (2) は少なくとも1つの密集区間が存在することを要求する。

条件 (1) の意味

全体を通じては低頻度のパターン。従来の FIM では閾値以下として棄却される。

条件 (2) の意味

特定の時間区間内ではサポートが密集閾値を超え、局所的に高頻度に出現する。

3.4 弱反単調性定理

定理 1: 稀少性条件の弱反単調性（上方閉性）

P ⊆ Q ならば supG(Q) ≤ supG(P)。すなわち、パターン P がグローバルに稀少でない（supG(P) ≥ θrare）ならば、P の部分集合 P' ⊆ P もまた稀少でない。

P ⊆ Q ⇒ supG(Q) ≤ supG(P)

対偶: supG(P) ≥ θrare ⇒ ∀P' ⊆ P, supG(P') ≥ θrare

含意: 稀少条件は上方閉（upward-closed）であり、スーパーセット方向に伝播する。これは従来の反単調性（サブセット方向）とは逆方向だが、候補生成時に「全部分集合がグローバル頻出ならばそのパターンも頻出であり、RDP ではない」という枝刈りに利用できる。具体的には、全ての k-1 部分集合が θrare 以上のサポートを持つ k-アイテムセットは候補から除外できる。

従来の反単調性（下方閉）

P が非頻出 ⇒ P の全スーパーセットも非頻出。頻出パターンの候補を下から上へ枝刈り。FIM の基盤。

弱反単調性（上方閉）

P が非稀少 ⇒ P の全サブセットも非稀少。稀少パターンの候補を上から下へ枝刈り。RDP の基盤。

命題: 枝刈り効率

θrare が十分低い場合、大多数のアイテムセットはグローバル頻出であるため枝刈り対象となり、候補空間は大幅に削減される。実験では候補アイテムセットの 82% がこの枝刈りにより除去された。

4. 提案手法: RDP-Miner

RDP-Miner は、RDP を効率的に発見する3ステップパイプラインである。グローバル稀少候補の抽出、Apriori-window による局所密集区間検出、および RDP 条件によるフィルタリングを統合する。

Step 1: グローバル稀少候補抽出

全アイテムのグローバルサポートを計算し、supG(i) < θrare を満たすアイテムを稀少アイテムとして抽出する。稀少アイテムのみからなるアイテムセットを候補として生成。弱反単調性により、全部分集合がグローバル頻出のアイテムセットを枝刈りする。

2

Step 2: Apriori-window による局所密集区間検出

Step 1 で抽出された稀少候補に対して、Apriori-window のスライディングウィンドウ機構を適用する。各候補パターンについて、ウィンドウサイズ W のスライディングウィンドウを移動させ、局所サポート supW(P, t) ≥ θdense を満たす連続区間（密集区間）を全て検出する。

入力: 稀少候補 P, トランザクション列 D, W, θdense
出力: P の密集区間リスト {[s1, e1], [s2, e2], ...}

ウィンドウ内サポートはインクリメンタルに更新（O(1) per step）し、全体として O(N) で全密集区間を列挙する。

3

Step 3: RDP フィルタリングと出力

Step 2 で密集区間が検出された候補のみを RDP として出力する。密集区間が1つも存在しない候補は棄却される。出力は (パターン, 密集区間リスト, グローバルサポート) の3つ組からなる。

RDP(P) = { (P, [si, ei], supG(P)) | supG(P) < θrare ∧ ∀t ∈ [si, ei]: supW(P, t) ≥ θdense }

実用的計算量

弱反単調性による枝刈りにより候補数 |C| は劇的に削減される。θrare が低いほど稀少アイテム数が少なくなり、候補空間は小さくなる。実験では200アイテム中の稀少アイテムは数十程度であり、候補ペア数は数百のオーダーに抑えられた。これにより100Kトランザクションに対して1秒未満で処理が完了する。

従来の FIM（Apriori, FP-Growth）はグローバル稀少パターンを棄却するため Recovery 0%。RP-tree は稀少パターンを発見できるが密集区間を出力せず、大量の FP を含む。LPFIM は局所密集を検出するがグローバル頻出パターンも出力するため精度が低い。RDP-Miner のみが Recovery 100% かつ FP 0 を達成し、密集区間も出力する。

比較のポイント

RDP-Miner の優位性は、稀少性と局所密集性の同時フィルタリングにある。RP-tree は稀少パターンを網羅的に列挙するが、その中で時間的に密集するものを区別する機能を持たない。結果として8,920パターンという大量の出力を返し、ユーザーが手動で選別する必要がある。LPFIM は局所密集パターンを検出するが、グローバルに頻出なパターンも含むため、「グローバルに稀少」という RDP の本質的特性を捉えられない。RDP-Miner はこれら二つの条件を統合することで、真に情報量の高いパターンのみを出力する。

6. 結論

本稿では、グローバルに稀少だが局所的に密集するアイテムセットという新しいパターンクラス 稀少密集パターン（RDP）を定義し、その効率的な発見アルゴリズム RDP-Miner を提案した。

主要な成果

  • 新パターンクラスの確立: 従来の FIM が見逃すパターン空間（グローバル稀少 × 局所密集）を形式的に定義し、その実用的重要性を示した
  • 弱反単調性による枝刈り: 稀少性条件の上方閉性を証明し、候補空間を82%削減する効率的な枝刈り戦略を実現した
  • 完全な正解回復: 合成データの全シード・全パラメータ設定で GT Recovery 100% を達成（FP = 0）
  • 実データでの有効性: Online Retail データで従来手法比99%の出力削減を実現し、ドメイン知識と整合するパターンを発見した
  • 線形スケーラビリティ: 100K トランザクションで1秒未満、100万トランザクションで約9秒の処理時間

制約と限界

  • パラメータ設定: θrare と θdense の適切な設定はドメイン知識に依存する。自動チューニング手法の開発が望まれる。特に θrare が高すぎると偽陽性が増加し、θdense が高すぎると弱い信号を見逃す。
  • アイテムセット次数: 現在の実装はペアワイズ（2-アイテムセット）に最適化されている。高次アイテムセットへの拡張では候補爆発の制御が課題となる。閉パターンや極大パターンの概念を活用した候補削減が必要である。
  • 統計的有意性: 密集区間の統計的有意性は未検証であり、偶然の密集と真の RDP の区別には追加の検定が必要。Paper A の置換検定フレームワークとの統合が一つの解決策である。
  • 非定常性: 非定常データ（トレンドや季節性を持つ）ではベースラインサポートの推定が課題となる。時変閾値の導入が今後の方向性として考えられる。
  • 重複区間の扱い: 同一パターンが複数の密集区間を持つ場合、区間間の関係（周期性、クラスタリング等）の分析は現在のフレームワークの範囲外である。

今後の課題

  1. イベント帰属との統合: Paper A のイベント帰属パイプラインとの統合により、「なぜ」RDP が特定区間に密集するかの原因分析を実現する。RDP の密集開始を変化点として扱い、外部イベントへの帰属を検定する。
  2. 統計的有意性検定: 置換検定の導入により、偶然の密集と統計的に有意な RDP を区別する。帰無分布としてサポート時系列の円形シフトを使用し、密集区間の出現確率を推定する。
  3. 高次アイテムセット: 閉パターンや極大パターンの概念を活用した候補削減により、3-アイテムセット以上への効率的な拡張を実現する。弱反単調性の高次拡張も理論的に検討する。
  4. オンライン検出: ストリーミング環境でのインクリメンタルな RDP 検出アルゴリズムの設計。新規トランザクション到着時に密集区間をリアルタイムで更新する。
  5. 応用展開: サイバーセキュリティ（Paper M との連携）、医療データ（稀少疾患の共起パターン）、金融市場（異常取引パターン）など高インパクト領域への応用を推進する。
