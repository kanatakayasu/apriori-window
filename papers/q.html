Q

密集区間検出のための
トランザクション基盤モデル：
アーキテクチャ設計と概念実証

基盤モデル（Foundation Model）の枠組みをトランザクションデータに初めて適用。 Masked Transaction Modeling（MTM）と Dense Interval Prediction（DIP）の 2段階事前学習を提案し、概念実証として TransFM アーキテクチャを設計する。

Wave 5 On Hold 2-Stage Pretraining Transformer マルチホット埋め込み 概念実証 NeurIPS / KDD

ステータス: On Hold（概念実証段階）

本論文は Wave 5 の探索的研究であり、現時点では概念実証（PoC）段階にある。DIP ファインチューンの AUC が 0.497（ランダムレベル）であり、密集区間予測タスクとしての実用性は未確立。今後の大規模事前学習データセットの構築と、アーキテクチャの根本的見直しが必要である。

1. はじめに

基盤モデル（Foundation Model）は、NLP（BERT, GPT）、コンピュータビジョン（ViT, DINO）、時系列予測（TimeGPT, PatchTST）において、大規模事前学習と下流タスクへのファインチューニングのパラダイムを確立した。このパラダイムは、ドメイン固有の少量データでも高い汎化性能を発揮する点で、従来のタスク特化型モデルに対する優位性を示している。

しかし、トランザクションデータに対する基盤モデルは未だ提案されていない。トランザクションデータは、可変長のアイテム集合が時間軸上に並ぶ固有の構造を持ち、自然言語や連続時系列とは根本的に異なる表現空間を必要とする。

研究動機: なぜトランザクション基盤モデルか？

  • 統一的表現学習: 頻出パターンマイニング、密集区間検出、異常検知など、複数のタスクに転移可能な表現を一度の事前学習で獲得
  • ドメイン横断転移: 小売データで事前学習し、医療トランザクションや Web ログに転移する可能性
  • 少量データ問題: ドメイン固有の密集区間ラベルは高コストであり、教師なし事前学習によるラベル効率の向上が期待される
  • 暗黙的パターン知識: 事前学習済みモデルがアイテム共起構造を暗黙的に学習し、従来のサポートカウントを超えた密集検出を可能にする

本稿では、トランザクション基盤モデル TransFM（Transaction Foundation Model）のアーキテクチャを設計し、MTM（Masked Transaction Modeling）と DIP（Dense Interval Prediction）の2段階事前学習目的関数を提案する。概念実証として、合成データ上での MTM 収束性と Apriori-window との整合性を確認する。

本研究の貢献

  1. トランザクション埋め込み設計: マルチホットベクトルとフーリエ基底による時間位置符号化を組み合わせた、トランザクションデータ固有の埋め込み層を設計する。
  2. 2段階事前学習目的関数: MTM（マスクされたトランザクションの復元）と DIP（密集区間の予測）を段階的に学習する事前学習フレームワークを定式化する。
  3. 概念実証: MTM 収束性（損失 4.1% 削減）と Apriori-window との密集区間整合性（IoU 0.966）を確認し、TransFM の基本的な実現可能性を示す。

2. 関連研究

2.1 基盤モデル

BERT (Devlin et al., 2019) は Masked Language Modeling（MLM）による双方向事前学習を提案し、NLP の基盤モデルパラダイムを確立した。GPT (Radford et al., 2018; Brown et al., 2020) は自己回帰生成による大規模言語モデルを実現した。ViT (Dosovitskiy et al., 2021) は Transformer を画像に適用し、MAE (He et al., 2022) はマスク自己符号化器による視覚表現学習を提案した。

これらの基盤モデルは連続値（テキストトークン、画素値）を対象とするが、トランザクションデータのマルチホット集合構造（1トランザクション内の複数アイテム同時出現）には直接適用できない。

2.2 時系列基盤モデル

TimeGPT (Garza & Mergenthaler-Canseco, 2023) は時系列予測の基盤モデルとして、大規模時系列コーパスでの事前学習を実現した。PatchTST (Nie et al., 2023) はパッチ化された時系列入力による効率的な Transformer 学習を提案した。Lag-Llama (Rasul et al., 2024) は確率的時系列予測に LLM アーキテクチャを適用した。Chronos (Ansari et al., 2024) は時系列をトークン化し、言語モデルとして学習する手法を提案した。

これらは連続値時系列を対象としており、離散的なマルチホット集合の時系列であるトランザクションデータとは入力空間が根本的に異なる。

2.3 トランザクション表現学習

トランザクションデータの表現学習に関する研究は限られている。Item2Vec (Barkan & Koenigstein, 2016) は Word2Vec をアイテム推薦に適用したが、時間構造を考慮しない。BERT4Rec (Sun et al., 2019) はユーザ行動列にBERTを適用したが、バスケット内の同時購買構造を扱わない。本研究はトランザクションの集合構造と時間構造の両方を同時にモデル化する初の試みである。

3. 問題定義

3.1 トランザクション埋め込み

トランザクション列 D = [d1, ..., dN]（各 dt ⊆ I）を、連続ベクトル列に変換する埋め込み関数 f: 2I → Rd を設計する。

定義 1: マルチホット埋め込み

トランザクション dt のマルチホット表現 mt ∈ {0,1}|I| に対し、埋め込みベクトルを以下で定義する。

ht = Wproj · mt + pt
pt = [sin(t/100002i/d), cos(t/100002i/d)]i=0..d/2

Wproj ∈ Rd×|I| はアイテム射影行列、pt はフーリエ基底による時間位置符号化。

3.2 事前学習目的関数

Stage 1: MTM（Masked Transaction Modeling）

LMTM = -Et~M[log P(dt | D\M)]

トランザクション列の一部をマスクし、コンテキストから復元する。BERT の MLM をトランザクション単位に拡張。マスク率 15%。

Stage 2: DIP（Dense Interval Prediction）

LDIP = -EP,t[yPt log σ(fDIP(ht, P))]

各時点・各パターンに対して「密集区間内か否か」を二値分類。Apriori-window の出力を疑似ラベルとして使用。

3.3 下流タスク

想定される下流タスク

  • 密集区間検出: 事前学習済み TransFM を密集区間の境界検出にファインチューン
  • 異常トランザクション検出: 復元誤差の高いトランザクションを異常として検出
  • パターン予測: 将来のトランザクションに出現するアイテム集合を予測
  • ドメイン転移: 小売データで事前学習し、医療・サイバー等に転移

4. TransFM アーキテクチャ

1

マルチホット埋め込み層

各トランザクションをマルチホットベクトルに変換し、線形射影で d 次元空間に埋め込む。フーリエ基底の位置符号化を加算し、時間的位置情報を注入する。バッチ正規化により学習安定性を確保。

2

Transformer エンコーダ

L 層の Transformer エンコーダブロックでトランザクション列のコンテキスト表現を学習。Multi-Head Self-Attention により、時間的に離れたトランザクション間の依存関係を捕捉する。

3

Stage 1: MTM 事前学習

トランザクション列の15%をランダムにマスクし、マスク位置のアイテム集合を復元する目的関数で学習。アイテム共起構造と時間的パターンの汎用表現を獲得する。

4

Stage 2: DIP ファインチューン

Apriori-window の出力（密集区間ラベル）を疑似ラベルとして、各時点が密集区間内かを予測する二値分類ヘッドを追加学習。事前学習済み表現を密集区間タスクに特化させる。

DIP の課題

DIP AUC が 0.497 とランダムレベルであり、密集区間の予測タスクとしてはモデルが有効に学習できていない。IoU 0.966 は MTM 表現の品質を反映するものであり、DIP 自体の予測性能ではない。

考えられる原因: (1) 事前学習データの規模不足、(2) DIP ヘッドの設計が不適切、(3) 密集区間ラベルの分布が極端に偏っている（正例が少ない）。

6. 議論

6.1 概念実証の意義と限界

確認できたこと

  • MTM による事前学習は収束する
  • フーリエ位置符号化が時間構造の捕捉に有効
  • 事前学習済み表現は Apriori-window と高い整合性（IoU 0.966）を示す
  • 2段階学習フレームワーク自体は実装可能

未解決の課題

  • DIP AUC がランダムレベル（根本的な設計見直しが必要）
  • 大規模データでの事前学習未実施
  • ドメイン転移の実証なし
  • 計算コストと Apriori-window 直接実行の比較未実施

6.2 Paper E との関係

Paper E の Structure-Aware Top-k は密集区間の効率的な列挙を扱うが、TransFM はこれをニューラルネットワークで近似する試みである。Paper E の Top-k 構造をDIPの疑似ラベル生成に活用する設計が今後有望と考えられる。

6.3 今後の方向性

ロードマップ

  1. Phase 1: 大規模トランザクションコーパスの構築（Retail + Dunnhumby + Online Retail の統合）
  2. Phase 2: スケーラブルな MTM 事前学習（d=512, L=12）
  3. Phase 3: DIP ヘッドの再設計（セグメンテーション型 or 変化点検出型）
  4. Phase 4: ドメイン転移実験（小売 → 医療、小売 → サイバー）

7. 結論

本稿では、トランザクションデータに対する基盤モデル TransFM のアーキテクチャ設計と、MTM + DIP の2段階事前学習目的関数を提案した。概念実証レベルの実験により、MTM の収束性と Apriori-window との高い整合性（IoU 0.966）を確認した。

主要成果のまとめ

  • 新規性: トランザクションデータに対する基盤モデルの初の提案
  • マルチホット埋め込み: 集合構造と時間構造を同時にモデル化する埋め込み設計
  • MTM 収束: 損失 4.1% 削減、事前学習の実現可能性を確認
  • IoU 0.966: Apriori-window との高い整合性
  • DIP 課題: AUC 0.497 でありランダムレベル、根本的見直しが必要

本研究は探索的な概念実証であり、大規模事前学習とDIP再設計を経て、トランザクション基盤モデルの実用化への道筋を示すことが今後の課題である。
