# new_apriori_window アルゴリズム 要件定義（改良版）

## 1. 目的
`new_apriori_window` に、既存 `apriori_window` の移動ウィンドウ法を拡張した密集区間検出アルゴリズムを実装するための要件を定義する。

## 2. スコープ
- 対象: トランザクション列に対するアイテム(またはアイテムセット)の密集区間検出
- 範囲外: UI/可視化、評価指標の実装

## 3. 用語・定義
- トランザクション: 時系列で並ぶ取引/イベント。位置は整数インデックスで扱う。
- 出現位置列: 対象アイテム(アイテムセット)が出現したトランザクションの昇順インデックス列 `t_1 < ... < t_m`。
- ウィンドウ: 左端 `l` から `l + W` までの閉区間(右端含む)。
- `f(l)` (=`count(l)`): `t_i ∈ [l, l + W]` の個数。
- 密集区間: `count(l) >= θ` を満たす **ウィンドウ左端 `l` の集合**として表現する。本仕様では、密集区間は `l` の連続区間（区間集合）で表す。
- 密集探索状態:
  - `in_dense`: 密集探索中かどうか。
  - `start`: 密集探索開始時点のウィンドウ左端(定義は 5.4 Case C を参照)。
  - `end`: 密集探索中に観測される **左端 `l` の最大値**。

### 密集区間の厳密定義
区間 `[s, e]` を **「左端 `l` の連続区間」** として表現し、以下を満たすとき密集区間とする。

1. ∀left 条件(区間内のどこに置いても密であること)  
   `l` を整数とし、`l = s, s+1, ..., e` のすべてで `f(l) >= θ`。
2. 最大性  
   隣接する出現位置へ拡張すると条件 3 を満たさない。  
   - `l = s-1` のウィンドウは密集区間ではない。  
   - `l = e+1` のウィンドウは密集区間ではない。

## 4. 入力・出力
### 入力
- トランザクション列
- パラメータ
  - `W`: ウィンドウ幅
  - `θ`: 出現回数閾値(設定ファイルでは `min_support` をそのまま回数として扱う)
  - `stride_adjust`: 余剰分による右スライドの調整ルール(本仕様では `surplus` に基づく)

### 出力
- `apriori_window` 既存仕様に準拠した CSV 形式で出力する
  - 出力先: `settings.json` の `output_files` で指定したパス
  - ヘッダー: `pattern_components,pattern_gaps,pattern_size,intervals_count,intervals`
  - 1行1アイテムセット
  - `pattern_components`: `"[{a, b, c}]"` 形式(文字列)
  - `pattern_gaps`: `"[]"` 固定
  - `pattern_size`: アイテム数
- `intervals_count`: 密集区間数
- `intervals`: `"(s,e);(s,e)"` のセミコロン区切り(空の場合は空文字)
  - `s,e` は **ウィンドウ左端 `l` の区間**を表す
  - 出力対象は **アイテム数が2以上** のみ
  - 出力順は **アイテム数の降順** とする
  - 例:
    - `"[{0, 1}]", "[]", 2, 2, "(0,3);(10,25)"`

## 5. アルゴリズム仕様
### 5.1 前処理
1. 対象アイテム(アイテムセット)の出現位置列 `timestamps` を生成する。
2. アイテム数1の候補区間は「探索対象の全区間」を1つとして与える。

### 5.2 Apriori 段階的探索
1. アイテム数 1 の候補集合を作る。
2. 各候補について `compute_dense_intervals` 相当の処理を行い、密集区間が 1 つ以上ある候補のみを頻出候補として保持する。
3. `k -> k+1` の候補生成を行う。
   - `k` の頻出候補同士を join して `k+1` の候補を作る。
   - Apriori 性質により、`k+1` の候補の全 `k` 部分集合が頻出候補に含まれない場合は pruning する。
4. 各候補アイテムセット `X` に対して、構成要素アイテムの **「密になる左端 `l` の連続区間」** を計算し、その積集合を `X` の候補区間 `C(X)` とする。
   - `S(A) = { l | f_A(l) >= θ }` を「`l` が密になる集合」とし、`S(A)` の連結成分を区間表現する。
   - 例: `X = {A, B}` のとき、`C(X) = S(A) ∩ S(B)`。
   - アイテム数2以上のときは、`C(X)` のうち **長さが `W` 以上**の区間のみを候補として残す。
   - `C(X)` が空なら `X` は探索対象外とする。
5. 候補が空になるまで `k` を増やして繰り返す。
6. 以下の 5.3 以降は、**候補アイテムセットが与えられた場合の密集区間抽出手順**である。

### 5.3 初期化
- 候補区間リスト `C(X)` を用意する。
  - アイテム数1: `C(X)` は探索対象の全区間を1つだけ持つ。
  - アイテム数2以上: `C(X)` は密集区間の積集合から長さ `W` 以上の区間のみ残す。
- `l` は各候補区間の左端から開始する。
- `in_dense = false`
- `start = None`
- `end = None`

### 5.4 メインループ
各候補区間 `[c_s, c_e]` ごとに、`l` を `c_s` から開始して以下を繰り返す。

1. `count = count(l)` を計算する。
2. `last_in_window = max { t_i | t_i <= l + W }` を求める(存在しない場合は `None`)。

#### Case A: `count < θ`
- 密ではない。
- `in_dense == true` なら密集区間候補 `[start, end]` を確定する。
- `in_dense = false`
- `l` を次の出現位置に進める(次の `t_i`、存在しない場合は終了)。

#### Case B: `count == θ`
- 密である(閾値ちょうど)。
- `in_dense == false` なら探索を開始する:
  - `in_dense = true`
  - `start = w_{count-1-(surplus)} - W`
  - `end = l`
- ここで `start` を `w_{count-1-(surplus)} - W` とする理由:
  - `surplus = count - θ` なので、Case B では `surplus = 0`。
  - `w_{count-1-(surplus)}` は「右から (surplus+1) 個目の出現位置」を表す。
  - したがって Case B では `w_{count-1}` が右端(= `last_in_window`)になり、
    `[w_{count-1} - W, w_{count-1}]` のウィンドウでも `count >= θ` が成立する。
- 既に探索中なら `end = max(end, l)` を更新する。
- `l = l + 1` (1 トランザクションずつ検証)
- `count < θ` になった時点で探索を終了する(Case A で確定)。

#### Case C: `count > θ`
- 密である(余剰あり)。
- `in_dense == false` なら探索を開始する:
  - `in_dense = true`
  - `start = w_{count-1-(surplus)} - W`
  - `end = l`
- 既に探索中なら `end = max(end, l)` を更新する。
- 余剰分 `surplus = count - θ` を算出する。
- 現ウィンドウ内の出現位置列を左から `w_0, w_1, ..., w_{count-1}` とすると、
  - `l = w_surplus` に更新する(左側 `surplus` 個を捨てるストライド)。
- ここで `start` を `w_{count-1-(surplus)} - W` とする理由:
  - `w_{count-1-(surplus)}` は「右から (surplus+1) 個目の出現位置」を表す。
  - 現在のウィンドウ `[l, l+W]` には `count` 個の出現位置が含まれている。
  - `w_{count-1-(surplus)}` を右端にする窓 `[w_{count-1-(surplus)} - W, w_{count-1-(surplus)}]` を取ると、
    少なくともその右側に `surplus` 個、合計で `θ` 個の出現位置が必ず含まれるため、
    この窓でも `count >= θ` が成立する。
- 次のループへ進む。

### 5.5 終了処理
各候補区間の探索終了時に `in_dense == true` なら、候補区間 `[start, end]` を確定する。

### 5.6 出力
確定した区間を、4章の CSV 形式で 1 行 1 アイテムセットとして出力する。

## 6. 例外・境界条件
- 出現位置列が空の場合は密集区間なし。
- `last_in_window` が存在しない場合は Case A と同様に扱う。
- `W >= 1`, `θ >= 1` を前提とする。

## 7. 非機能要件
- 同一入力に対して決定的な出力を返す。
- 大規模データに対しても現実的な計算量で動作すること。
