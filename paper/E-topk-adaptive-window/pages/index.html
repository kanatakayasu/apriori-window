Parameter-Free Dense Interval Mining: Top-k Patterns with Adaptive Window Sizes

Eliminating the need for window size and support threshold parameters through multi-scale analysis and branch-and-bound top-k mining.

VLDB / ICDM 2027 Phase 6 Complete

Motivation

The Problem: Dense interval mining requires two critical parameters — window size W and support threshold θ — whose tuning demands domain expertise and costly trial-and-error. Different parameter choices reveal fundamentally different patterns.
Our Solution: Replace (W, θ) with a single intuitive parameter k (number of desired patterns), and automatically analyze across multiple window scales using a dyadic hierarchy.

Key Contributions

Dense Coverage Score

A ranking criterion that jointly captures temporal extent and density magnitude, enabling meaningful comparison across patterns and scales.

Dyadic Scale Hierarchy

Multi-scale analysis at window sizes W, 2W, 4W, ... with proportional thresholds, covering the full range of temporal resolutions.

Scale-Space Dense Ridges

Connected components in the scale-position plane that identify patterns with density persisting across multiple scales — scale-invariant evidence of genuine temporal concentration.

Branch-and-Bound Pruning

Exploits anti-monotonicity of DCS across both the itemset lattice and scale hierarchy for provably correct early termination.

Algorithm Pipeline

Build Item Maps → Multi-Scale DCS → Top-k Heap → B&B Pruning → Ridge Detection
MSDCS(P) = Σℓ ωℓ · DCS(P, Wℓ, θℓ)   where   Wℓ = 2ℓ · W0
