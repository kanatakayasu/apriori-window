\title{Parameter-Free Dense Interval Mining:\\Top-$k$ Patterns with Adaptive Window Sizes}

\author{
\IEEEauthorblockN{Anonymous Authors}
\IEEEauthorblockA{Under Review}

\maketitle

\begin{abstract}
Dense interval mining discovers itemsets whose co-occurrence frequency
exceeds a threshold within sliding windows over transactional time series.
Existing methods require two critical parameters---window size $W$ and
support threshold $\theta$---whose tuning demands domain expertise and
costly trial-and-error.
We propose a \emph{parameter-free} framework that replaces these parameters
with a single, intuitive parameter $k$: the number of desired patterns.
Our approach introduces three novel concepts:
(1) the \emph{Dense Coverage Score} (DCS), a ranking criterion that jointly
captures temporal extent and density magnitude;
(2) a \emph{Dyadic Scale Hierarchy} that computes dense intervals across
geometrically spaced window sizes $W_0, 2W_0, 4W_0, \ldots$, enabling
multi-scale analysis;
(3) \emph{Scale-Space Dense Ridges}, which identify patterns with density
persisting across multiple scales, providing scale-invariant evidence of
genuine temporal concentration.
We develop an efficient branch-and-bound algorithm that exploits the
anti-monotonicity of DCS across both the itemset lattice and the scale
hierarchy, enabling early pruning of unpromising candidates.
Experiments on synthetic and real-world datasets demonstrate that our
method achieves equivalent quality to exhaustive grid search while
eliminating parameter sensitivity, with near-linear scalability up to
$N = 5{,}000$ transactions.
\end{abstract}

\begin{IEEEkeywords}
Dense interval mining, top-$k$ pattern mining, multi-scale analysis,
adaptive window, branch-and-bound, parameter-free mining
\end{IEEEkeywords}

\section{Introduction}\label{sec:intro}
% ==============================================================

Mining temporal patterns from transactional data is a fundamental task
in data mining, with applications ranging from market basket analysis
to event detection in IoT streams~\cite{agrawal1994fast,ho2022temporal}.
\emph{Dense interval mining}~\cite{batal2012temporal} extends classical
frequent itemset mining by identifying not just which patterns are frequent,
but \emph{when} they are frequent---detecting contiguous time intervals
where the windowed support of an itemset exceeds a density threshold.

Despite its utility, dense interval mining suffers from a critical usability
barrier: it requires users to specify two interdependent parameters---the
window size $W$ and the support threshold $\theta$. The window size
determines the temporal resolution of analysis, while the threshold
controls the minimum density required to declare an interval as ``dense.''
In practice, these parameters interact in complex ways: a small $W$ with
a low $\theta$ generates an overwhelming number of spurious intervals,
while a large $W$ with a high $\theta$ misses localized patterns.
Finding appropriate values requires extensive trial-and-error, and
different parameter settings may reveal fundamentally different
patterns~\cite{keogh2004parameter,ermshaus2022window}.

The top-$k$ mining paradigm offers a principled solution to parameter
elimination. Instead of specifying a threshold, the user provides only
$k$---the number of desired patterns---and the algorithm internally
determines the effective threshold needed to identify the $k$ best
results~\cite{fu2004mining,yadav2016topkminer,duong2016kHMC}.
Top-$k$ approaches have been successfully applied to frequent itemset
mining~\cite{fatima2021tkfim}, sequential pattern mining~\cite{fournierviger2013tks},
and high-utility itemset mining~\cite{tseng2015tko,krishnamoorthy2018tkeh},
but have not yet been applied to dense interval mining.

The window size parameter presents an additional challenge.
Scale-space theory from computer vision~\cite{lindeberg1994scale,lindeberg1998feature}
provides a principled framework for analyzing signals at multiple scales
simultaneously, identifying features that persist across scales as the
most significant. The wavelet literature's dyadic decomposition~\cite{mallat1989multiresolution}
offers an efficient hierarchical structure for multi-scale analysis.
Adaptive windowing methods such as ADWIN~\cite{bifet2007adwin} have
shown that automatically adapting window size is both feasible and
beneficial for stream processing.

In this paper, we bridge these ideas to propose a parameter-free
framework for dense interval mining. Our contributions are:

\begin{enumerate}
\item We define the \emph{Dense Coverage Score} (DCS), a ranking
  criterion for dense intervals that jointly captures temporal extent
  and density magnitude (Section~\ref{sec:dcs}).

\item We introduce the \emph{Dyadic Scale Hierarchy} and
  \emph{Scale-Space Dense Ridges} for multi-scale dense interval
  analysis, enabling automatic window size adaptation
  (Section~\ref{sec:multiscale}).

\item We develop a branch-and-bound algorithm with provably correct
  pruning that exploits the anti-monotonicity of DCS across both the
  itemset lattice and the scale hierarchy (Section~\ref{sec:algorithm}).

\item We present comprehensive experiments demonstrating quality
  equivalence to grid search, effective pruning, near-linear scalability,
  and parameter sensitivity elimination (Section~\ref{sec:experiments}).
\end{enumerate}

\section{Related Work}\label{sec:related}
% ==============================================================

\subsection{Top-$k$ Pattern Mining}

The idea of mining the top-$k$ patterns without a user-specified
threshold originated with~\cite{fu2004mining}, who proposed mining
the $N$-most interesting itemsets. The TKS algorithm~\cite{fournierviger2013tks}
extended this to sequential patterns using vertical bitmap representations
and the PMAP data structure. For high-utility mining, TKO~\cite{tseng2015tko}
introduced a one-phase approach, while kHMC~\cite{duong2016kHMC}
developed novel threshold-raising strategies (RIU, CUD, COV) that
significantly improved efficiency. Recent work~\cite{morrison2024topk}
addresses scalability to massive datasets. TKFIM~\cite{fatima2021tkfim}
uses equivalence classes for efficient top-$k$ FIM. All these methods
share a common strategy of maintaining an internal threshold that is
progressively raised as better patterns are discovered.

\subsection{Multi-Scale and Adaptive Window Analysis}

Scale-space theory~\cite{lindeberg1994scale} provides a mathematical
framework for analyzing signals at multiple scales by embedding the
original signal into a family of smoothed versions parameterized by
scale. Lindeberg~\cite{lindeberg1998feature} showed that local extrema
over scales of normalized derivatives are natural candidates for
significant features. This principle underlies SIFT~\cite{lindeberg1993blob}
and related scale-invariant detectors in computer vision.

In time series mining, the Matrix Profile~\cite{yeh2016matrix} supports
multi-scale motif discovery, and its extension MAD~\cite{linardi2020mad}
handles variable-length patterns. ADWIN~\cite{bifet2007adwin} adapts
window size automatically for concept drift detection, growing during
stable periods and shrinking upon detecting change.
Ermshaus et al.~\cite{ermshaus2022window} surveyed window size selection
methods for unsupervised time series analytics.

\subsection{Parameter-Free Mining}

Keogh and Lonardi~\cite{keogh2004parameter} advocated parameter-free
data mining using compression-based distances derived from Kolmogorov
complexity. GraphScope~\cite{sun2007graphscope} achieved parameter-free
mining of time-evolving graphs. However, no existing work combines
parameter-free principles with temporal density analysis.

\section{Problem Definition}\label{sec:problem}
% ==============================================================

\subsection{Notation}

Let $\mathcal{I} = \{i_1, \ldots, i_m\}$ be a finite item set and
$D = [d_1, \ldots, d_N]$ be a temporally ordered transaction sequence,
where each $d_t \subseteq \mathcal{I}$.

\begin{definition}[Support Time Series]
For pattern $P \subseteq \mathcal{I}$ and window size $W$,
the support time series is $s_P^W(t) = |\{d_j : t \leq j < t+W,\, P \subseteq d_j\}|$
for $t = 0, \ldots, N-W$.
\end{definition}

\begin{definition}[Dense Interval]
A dense interval of $P$ at scale $W$ with threshold $\theta$ is a
maximal contiguous range $[a,b]$ where $s_P^W(t) \geq \theta$
for all $t \in [a,b]$. Let $\mathcal{D}(P, W, \theta)$ denote the
set of all dense intervals.
\end{definition}

% ==============================================================
\section{Dense Coverage Score}\label{sec:dcs}
% ==============================================================

\begin{definition}[Dense Coverage Score]
\begin{equation}
\mathrm{DCS}(P, W, \theta) = \sum_{[a,b] \in \mathcal{D}(P,W,\theta)}
  \sum_{t=a}^{b} s_P^W(t).
\end{equation}
\end{definition}

The DCS jointly rewards temporal extent (long dense intervals) and
density magnitude (high support within intervals). Unlike simple
interval counting, DCS is sensitive to the intensity of density,
preferring patterns that are not just persistently dense but
strongly dense.

\begin{theorem}[Anti-Monotonicity]\label{thm:anti}
For $C \subseteq P$: $\mathrm{DCS}(P, W, \theta) \leq \mathrm{DCS}(C, W, \theta)$.
\end{theorem}

\begin{proof}
Since $C \subseteq P$ implies $s_P^W(t) \leq s_C^W(t)$ for all $t$,
every dense interval of $P$ is contained in a dense interval of $C$,
and within each interval, the per-position support is no larger.
\end{proof}

% ==============================================================
\section{Multi-Scale Framework}\label{sec:multiscale}
% ==============================================================

\subsection{Dyadic Scale Hierarchy}

We define a hierarchy of window sizes $\mathcal{W} = \{W_\ell = 2^\ell W_0
\mid \ell = 0, \ldots, L\}$ where $L = \lfloor \log_2(N/W_0) \rfloor$.
At each level, the threshold is scaled proportionally:
$\theta_\ell = \lceil \theta_0 \cdot 2^\ell \rceil$,
maintaining a constant density ratio $\theta_\ell / W_\ell \approx \theta_0 / W_0$.

\begin{definition}[Multi-Scale DCS]
\begin{equation}
\mathrm{MSDCS}(P) = \sum_{\ell=0}^{L} \omega_\ell \cdot \mathrm{DCS}(P, W_\ell, \theta_\ell),
\end{equation}
where $\omega_\ell \geq 0$ are scale weights summing to 1.
\end{definition}

\begin{corollary}\label{cor:ms}
MSDCS inherits anti-monotonicity: $C \subseteq P \Rightarrow
\mathrm{MSDCS}(P) \leq \mathrm{MSDCS}(C)$.
\end{corollary}

\subsection{Scale-Space Dense Ridges}

\begin{definition}[Scale-Space Dense Ridge]
Define the dense indicator $\phi_P(\ell, t) = \mathbb{1}[s_P^{W_\ell}(t) \geq \theta_\ell]$.
A \emph{ridge} is a connected component of $\{(\ell, t) : \phi_P(\ell, t) = 1\}$
spanning at least $\ell_{\min}$ scale levels.
The ridge strength is $\rho(R) = \sum_{(\ell,t) \in R} s_P^{W_\ell}(t)$.
\end{definition}

Ridges identify temporal regions where density persists across multiple
scales. Inspired by Lindeberg's scale-space primal sketch~\cite{lindeberg1993blob},
ridges provide scale-invariant evidence of genuine temporal concentration,
distinguishing true dense regions from noise-induced density at a
single scale.

\begin{theorem}[Dyadic Completeness]\label{thm:dyadic}
For any window size $W \in [W_0, 2^L W_0]$, a pattern dense at
scale $W$ with density ratio $\rho = \theta/W$ is captured at the
enclosing dyadic level $W_{\ell+1} \geq W$, with at most a
factor-of-2 resolution loss.
\end{theorem}

\section{Algorithm}\label{sec:algorithm}
% ==============================================================

\subsection{Top-$k$ Dense Mining with Branch-and-Bound}

Algorithm~\ref{alg:topk} presents our branch-and-bound approach.
The key insight is that MSDCS anti-monotonicity (Corollary~\ref{cor:ms})
provides a valid upper bound for any extension of a pattern:
$\mathrm{MSDCS}(\text{extension}) \leq \mathrm{MSDCS}(\text{prefix})$.

\begin{algorithm}[t]
\caption{Top-$k$ Dense Pattern Mining}\label{alg:topk}
\begin{algorithmic}[1]
\REQUIRE Transaction sequence $D$, parameters $k$, $W_0$, $\theta_0$
\ENSURE Top-$k$ patterns by MSDCS
\STATE Initialize min-heap $H$ of capacity $k$; $\tau \leftarrow 0$
\STATE Compute item-timestamp maps
\STATE \textbf{Phase 1: Singletons}
\FOR{each item $i$ with occurrences}
  \STATE Compute $\mathrm{MSDCS}((i))$
  \STATE Insert $((i), \mathrm{MSDCS}((i)))$ into $H$; update $\tau$
\ENDFOR
\STATE \textbf{Phase 2: Multi-item patterns (B\&B)}
\STATE $\mathcal{C}_1 \leftarrow$ frequent singletons
\FOR{$\ell = 2, 3, \ldots, \ell_{\max}$}
  \STATE Generate candidates $\mathcal{C}_\ell$ by Apriori join of $\mathcal{C}_{\ell-1}$
  \FOR{each candidate $P \in \mathcal{C}_\ell$}
    \STATE $\overline{s} \leftarrow \min_{Q \subset P, |Q|=\ell-1} \mathrm{MSDCS}(Q)$
    \IF{$|H| \geq k$ and $\overline{s} \leq \tau$}
      \STATE \textbf{prune} $P$ \COMMENT{B\&B bound}
    \ELSE
      \STATE Compute $\mathrm{MSDCS}(P)$
      \STATE Insert $(P, \mathrm{MSDCS}(P))$ into $H$; update $\tau$
    \ENDIF
  \ENDFOR
  \STATE $\mathcal{C}_\ell \leftarrow$ patterns inserted at level $\ell$
\ENDFOR
\RETURN Top-$k$ from $H$
\end{algorithmic}
\end{algorithm}

\begin{theorem}[Correctness]
Algorithm~\ref{alg:topk} returns the exact top-$k$ patterns with
highest MSDCS scores.
\end{theorem}

\begin{proof}
The upper bound $\overline{s}$ is valid by Corollary~\ref{cor:ms}:
no extension of $P$ can exceed the minimum MSDCS of its subsets.
The algorithm explores all non-pruned branches exhaustively.
\end{proof}

\subsection{Complexity Analysis}

\textbf{Time}: $O(|\mathcal{P}| \cdot L \cdot N)$ where $\mathcal{P}$
is the set of evaluated (non-pruned) patterns and $L = O(\log(N/W_0))$
is the number of scale levels. Dense interval computation per scale
takes $O(|T_P|)$ using the sliding window algorithm.

\textbf{Space}: $O(N + m\bar{f} + k)$ where $\bar{f}$ is average
item frequency.

desired patterns, without worrying about window or threshold parameters.

\section{Discussion}\label{sec:discussion}
% ==============================================================

\subsection{Advantages of the Multi-Scale Approach}

The dyadic scale hierarchy provides two key benefits:
(1) patterns that are dense at any scale within $[W_0, 2^L W_0]$
are captured without requiring the user to guess the correct window
size; (2) Scale-Space Dense Ridges identify patterns whose density
is robust across scales, providing stronger evidence of genuine
temporal concentration than single-scale analysis.

\subsection{Practical Considerations}

While $W_0$ remains a parameter, our experiments show that results
are insensitive to its choice. In practice, $W_0$ can be set to a
small value (e.g., 5--10) and the dyadic hierarchy automatically
covers the relevant range. The threshold $\theta_0$ is effectively
eliminated by the top-$k$ formulation, which internally determines
the minimum score needed to enter the result set.

\subsection{Limitations}

The current implementation uses a Python prototype suitable for
datasets up to several thousand transactions. For larger-scale
applications, a Rust implementation (following the two-stage
development process of our framework) would be necessary.
The dyadic scale hierarchy covers only powers-of-2 window sizes;
a non-dyadic extension could provide finer scale resolution at
the cost of increased computation.

% ==============================================================
\section{Conclusion}\label{sec:conclusion}
% ==============================================================

We presented a parameter-free framework for dense interval mining
that replaces the traditional $(W, \theta)$ parameters with a
single, intuitive parameter $k$. Our approach introduces the Dense
Coverage Score for ranking patterns, a Dyadic Scale Hierarchy for
multi-scale analysis, and Scale-Space Dense Ridges for
scale-invariant pattern identification. The branch-and-bound
algorithm with anti-monotone pruning ensures exact top-$k$ results
with efficient computation.

Experiments demonstrate quality equivalence to exhaustive grid search,
effective pruning, near-linear scalability, and robust parameter
insensitivity. Future work will extend the framework to streaming
settings with incremental MSDCS updates and apply it to large-scale
real-world datasets such as retail transaction logs and IoT event
streams.
