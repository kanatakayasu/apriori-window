\label{sec:supplement}

本補遺では，本文の実験で用いた合成データ生成の詳細，
パラメータ設定，および再現手順を補足する．

\subsection{合成データ生成モデル}
\label{sec:supp_data_gen}

ベース語彙 $|\mathcal{I}| = 200$ アイテム（ID: 1--200）を持つ
合成トランザクションデータを以下の手順で生成する．

\begin{enumerate}
\item 各トランザクション $d_t$（$t = 1, \ldots, N$）に対して，
アイテム $i \in \mathcal{I}$ をそれぞれ独立に確率 $p_{\text{base}} = 0.03$ で含める．
\item \textbf{Type A（語彙内ブースト）}:
植え込みパターン $P_k = \{a_k, b_k\}$（ベース語彙内アイテム，ID 1--200）と
対応するイベント $e_k = (\textit{id}_k, \textit{name}_k, t_{s,k}, t_{e,k})$ を定義する．
これらのアイテムはイベント期間外でもベースライン確率 $p_{\text{base}} = 0.03$ で出現する．
\item イベント活性期間 $[t_{s,k}, t_{e,k}]$ 内の各トランザクションに対して，
確率 $\beta$（ブースト係数）でパターン $P_k$ の全アイテムを同時挿入する．
\item \textbf{Type B（イベント非対応密集パターン）}:
いかなるイベントにも対応しないが，ランダムに選択された時間区間で
ベース語彙内アイテムペアのサポートをブーストする．
季節性・トレンドによる自然変動をシミュレートする．
\item \textbf{Type C（デコイイベント）}:
Ground Truth に含まれない偽イベントをランダムな位置に配置する．
デコイは植え込みアイテムをブーストしない．
\end{enumerate}

正解（Ground Truth）は（Type A 植え込みパターン $P_k$，対応イベント $e_k$）の
組として定義する．パイプラインの出力がこの組を正しく回復できるかを評価する．
Type B パターンのいかなるイベントへの帰属も偽陽性（FAR に計上）として扱う．

\subsection{実データの前処理}
\label{sec:supp_realdata}

EX5 で用いた実データは以下の通りである．

\begin{itemize}
\item \textbf{T10I4D100K}: IBM Quest Synthetic Data Generator で生成された
100{,}000 トランザクションのスパースなベンチマークデータ．
各トランザクションの平均アイテム数は約10．
\item \textbf{retail.txt}: ベルギーの小売店における約88{,}000件の
POS トランザクションデータ~\cite{fournier2017survey}．
\end{itemize}

いずれも1行1トランザクション形式（スペース区切り）で読み込み，
語彙内アイテムを用いた植え込み信号を追加して評価した．

\subsection{再現手順}
\label{sec:supp_reproduce}

全実験は以下のコマンドで再現可能である:

\begin{verbatim}
# EX1: 検出精度
python3 experiments/run_e1.py

# EX2: アブレーション
python3 experiments/run_e2.py

# EX3: 帰無データFDR制御
python3 experiments/run_e3.py

# EX5: 実データ検証
python3 experiments/run_e5.py

# EX6: Zipf頑健性
python3 experiments/run_e6.py
\end{verbatim}

実行環境: Python 3.11+，追加ライブラリ不要（標準ライブラリのみ使用）．
乱数は Python の \texttt{random.Random(seed)} を用いており，
同一 seed・同一パラメータで決定的に再現される．
