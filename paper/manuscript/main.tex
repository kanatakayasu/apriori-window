\begin{abstract}
既存の頻出パターンマイニングは主にグローバル頻度に依存するため，特定期間に集中して現れる局所的な密集パターンを十分に捉えにくい．
また，Gap 系手法は最大ギャップ閾値への感度が高く，パラメータ設定に結果が左右される．
この課題に対して DSAA2025 では，局所的な出現頻度の変動を厳密に捉えるApriori-window が提案され，最大ギャップ閾値に依存しない密集パターン抽出を実現した．
しかし同手法は，1 トランザクションを 1 アイテムセットとして扱う前提を置いており，同一時刻に複数の独立した購買イベント（バスケット）が含まれるデータでは，
バスケット間のアイテムが誤って共起として数えられる偽共起が発生する．
本稿はこの未解決課題に対して，DSAA2025 の枠組みを維持したままバスケット構造を扱える拡張手法（Basket-aware Apriori-Window）を提案する．
提案手法は，共起判定をトランザクション内からバスケット内へ置き換えることで，
偽共起パターン率（SPR: Spurious Pattern Rate）を原理的に 0 に抑える．
さらに，バスケット内共起においても Apriori の反単調性が成立することを示し，
従来と同様の候補枝刈りが可能であることを理論的に保証する．
合成データ実験では，まず Oracle との 34 ケース比較（E0）で
FP=0, FN=0 を確認し，実装厳密性を検証した．
さらに $\lambda_{\text{baskets}}$ sweep（E1）では，
従来法 GTSPR が 0.454 から 0.971 へ増加する一方，
提案手法は全条件で GTSPR=0 を維持した．
構造感度評価（E2）では，従来法 GTSPR が
$G=3 \rightarrow 50$ で 0.827 から 0.638，
$p_{\text{same}}=0.5 \rightarrow 0.9$ で 0.827 から 0.741 へ低下し，
Basket-aware Apriori-Window は同条件でも GTSPR=0 を維持した．
スケーラビリティ評価（E3）では，
$10^6$ トランザクションで Basket-aware Apriori-Window が 108.5 秒，
従来法が 145.9 秒（1.345 倍）となり，
規模拡大時に提案法の計算優位が増大することを確認した．
区間品質評価（E4）では，Basket-aware Apriori-Window の mean Jaccard が全条件で 1.000 となる一方，
従来法の interval purity は 0.075 から 0.018 へ低下し，
spurious interval rate は 0.735 から 0.945 の高水準を示した．
\end{abstract}
