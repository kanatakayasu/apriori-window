\begin{abstract}
頻出パターンマイニングは膨大な候補パターンを出力するため、
ユーザにとって結果の解釈が困難である。
本研究では、最小記述長（MDL: Minimum Description Length）原理を
密集区間構造に適用した新しいパターン要約手法
\textbf{Temporal Code Table（TCT）}を提案する。
TCTは各パターンに時間的有効区間を付与し、
\textbf{時間スコープ付き符号長}（Time-Scoped Code Length; TSCL）により
区間内での出現を効率的に符号化する。
合成データおよびベンチマーク実データでの実験において、
提案手法は候補パターンの87--93\%を削減しつつ、
KRIMP対比で5--18ポイント高い圧縮率を達成した。
これは密集区間の時間的局所性がデータ圧縮に有効であることを示す
初の体系的な実証である。
\end{abstract}

\textbf{キーワード:}
最小記述長, パターン要約, 密集区間, KRIMP, コードテーブル, 時間的局所性

\section{はじめに}\label{sec:intro}

頻出アイテムセットマイニング~\cite{agrawal1994fast}は
データマイニングの基盤技術であるが、
低い最小サポート閾値の下では爆発的な数のパターンが出力される。
この問題に対し、Vreekenら~\cite{vreeken2011krimp}は
MDL原理に基づくKRIMPアルゴリズムを提案し、
データを最も簡潔に記述するパターンセットを選択する枠組みを確立した。

しかし、KRIMPおよびその後継手法（SLIM~\cite{smets2012slim}、
StreamKrimp~\cite{van2014streamkrimp}）は
パターンの時間的構造を考慮しない。
実世界のトランザクションデータでは、
特定のパターンが特定の時間区間でのみ頻出する
\textbf{密集区間}（dense interval）現象が広く観察される。
例えば、小売データにおける季節商品の共起や、
キャンペーン期間中の購買パターンの変化がこれに該当する。

\begin{example}[動機付け]
1年間の小売データ（$N{=}365$日、30アイテム）を考える。
パターン$\{$マフラー, 手袋$\}$は冬季の60日間（$t{=}300$--$359$）のみ
高頻度で出現する。
KRIMPでは全365日に対するサポート率で符号長を計算するため、
$-\log_2(45/365) \approx 3.02$ビット/使用 となる。
一方、TCTでは冬季60日間のみを符号化対象とし、
$-\log_2(45/60) \approx 0.42$ビット/使用 で符号化できる。
この差$\Delta = 2.60$ビット$\times 45$回$= 117$ビットの圧縮利得が
密集区間の時間的局所性から直接得られる。
\end{example}

本研究では、密集区間の時間的局所性を活用した
MDLベースのパターン要約手法を提案する。
具体的な貢献は以下の3点である:

\begin{enumerate}[label=(\arabic*)]
  \item \textbf{Temporal Code Table（TCT）}:
    各パターンに時間的有効区間を付与したコードテーブルを定義し、
    区間内での符号化効率を向上させる（定義\ref{def:tct}）。
  \item \textbf{Time-Scoped Code Length（TSCL）}:
    パターンの区間内出現密度に基づく符号長関数を定義し、
    情報理論的に正当化された圧縮利得を導出する（定理\ref{thm:locality}）。
  \item \textbf{Dense Interval Compression（DIC）}:
    密集区間検出とMDL選択を統合した貪欲アルゴリズムにより、
    候補パターンの87--93\%を削減しつつ圧縮を達成する（定理\ref{thm:compression}）。
\end{enumerate}

\section{関連研究}\label{sec:related}

\subsection{MDLに基づくパターン選択}

MDL原理~\cite{rissanen1978modeling,grunwald2007minimum}は
モデルとデータの合計記述長を最小化する原理であり、
パターンマイニングにおけるモデル選択に広く応用されている。

\textbf{KRIMP}~\cite{vreeken2011krimp}は、
頻出パターンのコードテーブルを構築し、
Standard Cover Orderによる貪欲選択で
$L(\text{CT}) + L(D|\text{CT})$を最小化する。
\textbf{SLIM}~\cite{smets2012slim}はKRIMPを拡張し、
候補生成なしに直接コードテーブルを最適化する。
これらは静的なパターンセット選択であり、
パターンの時間的有効性を考慮しない。

\textbf{StreamKrimp}~\cite{van2014streamkrimp}は
ストリーミング環境でのコードテーブル更新を扱い、
データ分布の変化を検出するが、
個々のパターンの時間的有効区間を明示的にモデル化しない。
\textbf{Data-to-Model}アプローチ~\cite{dalleiger2022data}は
結果パターンセットから生成モデルを推論する枠組みを提案するが、
やはり時間軸は考慮されていない。

\subsection{情報理論的パターン選択}

MDL以外にも、情報理論に基づくパターン選択手法が提案されている。
\textbf{MTVパターン}~\cite{de2012mtv}は
最大エントロピー分布との差異を最大化するパターンを選択する。
\textbf{SQUISH}~\cite{mampaey2012squish}は
同様の原理をより効率的に実現する。
しかし、これらも時間的構造を利用しない点でKRIMPと同様の制約を持つ。

\subsection{時間的パターンマイニング}

密集区間検出は、アイテムセットの出現が
特定の時間区間に集中する現象を捉える手法である。
スライディングウィンドウに基づく密集区間検出~\cite{apriori_window}は
効率的なアルゴリズムを提供するが、
出力パターン数の制御はユーザの閾値設定に依存する。

時系列パターンマイニングの分野では、
\textbf{SPM}（Sequential Pattern Mining）~\cite{fournier2017survey}が
時間順序を持つパターンを扱うが、
密集区間の概念はSPMの枠組みには含まれない。
\textbf{TKS}~\cite{fournier2013tks}はTop-k系列パターンを
効率的に抽出するが、パターン数制御の基準がMDLではなくユーザ指定のkに依存する。

本研究は、密集区間検出の出力をMDL原理で要約するという
新しいアプローチを提案する。

\section{準備}\label{sec:prelim}

\subsection{記法}

$\mathcal{D} = \{T_1, T_2, \ldots, T_N\}$を
$N$個の\textbf{時刻付き}トランザクションからなるデータベースとする。
各$T_t$（$t = 1, \ldots, N$）はアイテム集合$\mathcal{I}$の部分集合であり、
$t$はトランザクションの時刻インデックスである。
アイテムセット$X \subseteq \mathcal{I}$の
トランザクション$T_t$における出現を$X \subseteq T_t$と書く。
$X$の全体サポートを$\text{sup}(X) = |\{t : X \subseteq T_t\}|$で定義する。

\begin{definition}[密集区間]\label{def:dense_interval}
ウィンドウサイズ$W \in \mathbb{Z}_{>0}$と密集閾値$\theta \in \mathbb{Z}_{>0}$に対し、
アイテムセット$X$の\textbf{密集区間}$[s, e]$（$1 \leq s \leq e \leq N$）とは、
以下を満たす\textbf{極大な}連続区間である:
\begin{equation}\label{eq:dense_def}
  \forall l \in [s, e{-}W{+}1],\quad
  |\{t \in [l, l{+}W{-}1] : X \subseteq T_t\}| \geq \theta.
\end{equation}
$X$の密集区間の集合を$\mathcal{I}_X$と書く。
\end{definition}

\begin{definition}[スコープ]\label{def:scope}
パターン$X$の密集区間集合$\mathcal{I}_X = \{[s_1,e_1], \ldots, [s_m,e_m]\}$に対し、
\textbf{スコープ}$\text{scope}(X) = \bigcup_{j=1}^{m} \{s_j, \ldots, e_j\}$を
有効トランザクション集合とする。
スコープサイズは$S_X = |\text{scope}(X)|$で表す。
\end{definition}

\subsection{KRIMPの概要}

KRIMPでは、コードテーブル$\text{CT}$は
パターンと符号の対応表であり、
データの二部記述長は以下で定義される:
\begin{equation}\label{eq:krimp}
  L(\text{CT}, \mathcal{D}) = L(\text{CT}) + L(\mathcal{D} | \text{CT})
\end{equation}
ここで$L(\text{CT})$はコードテーブル自体の記述長、
$L(\mathcal{D}|\text{CT})$はコードテーブルを用いた
データの記述長である。

カバー操作$\text{cover}(\text{CT}, T_t)$は、
Standard Cover Order（パターンサイズ降順、サポート降順）に従い
トランザクション$T_t$をパターンの非重複和集合で表現する。
パターン$X$の\textbf{使用回数}は
$\text{usage}(X|\text{CT}) = |\{t : X \in \text{cover}(\text{CT}, T_t)\}|$
と定義される。
各パターン$X$の符号長は使用頻度に基づくShannon符号:
\begin{equation}\label{eq:code_length}
  \ell(X | \text{CT}) = -\log_2 \frac{\text{usage}(X|\text{CT})}{\sum_{Y \in \text{CT}} \text{usage}(Y|\text{CT})}
\end{equation}

\section{提案手法}\label{sec:method}

\subsection{Temporal Code Table}

\begin{definition}[Temporal Code Table]\label{def:tct}
\textbf{Temporal Code Table（TCT）}は三つ組の集合
$\text{TCT} = \{(X_i, \mathcal{I}_i, c_i)\}_{i=1}^{k}$である。
ここで:
\begin{itemize}
  \item $X_i \subseteq \mathcal{I}$はパターン（$|X_i| \geq 2$）,
  \item $\mathcal{I}_i = \{[s_{i,1}, e_{i,1}], \ldots, [s_{i,m_i}, e_{i,m_i}]\}$は
    $X_i$の密集区間の集合（定義\ref{def:dense_interval}に従う）,
  \item $c_i = \text{usage}(X_i|\text{TCT})$は使用回数.
\end{itemize}
加えて、各単体アイテム$\{a\}$（$a \in \mathcal{I}$）は
時間スコープなし（$\mathcal{I}_a = \{[1,N]\}$）で
TCTに常に含まれる。
\end{definition}

\begin{definition}[Temporal Cover]\label{def:tcover}
TCTによるトランザクション$T_t$のカバー
$\text{cover}(\text{TCT}, T_t)$は以下の手順で計算される:
\begin{enumerate}
  \item $\text{uncovered} \leftarrow T_t$とする。
  \item Standard Cover Orderに従い各エントリ$(X_i, \mathcal{I}_i, c_i)$を走査:
    \begin{itemize}
      \item $X_i \subseteq \text{uncovered}$かつ$t \in \text{scope}(X_i)$ならば、
        $X_i$をカバーに追加し、$\text{uncovered} \leftarrow \text{uncovered} \setminus X_i$。
    \end{itemize}
  \item 残余$\text{uncovered}$中の各アイテムは単体パターンでカバーする。
\end{enumerate}
\end{definition}

カバー操作はKRIMPと同一であるが、
パターン$X_i$が使用可能な条件に
$t \in \text{scope}(X_i)$という\textbf{時間的制約}が追加される。
これにより、パターンの使用回数はスコープ内トランザクションに限定され、
区間内での出現密度が符号長に直接反映される。

\subsection{Time-Scoped Code Length}

\begin{definition}[Time-Scoped Code Length]\label{def:tscl}
TCTにおけるパターン$X$（$|X| \geq 2$）の
\textbf{時間スコープ付き符号長}を以下で定義する:
\begin{equation}\label{eq:tscl}
  L_{\text{ts}}(X|\text{TCT}) =
    -\log_2 \frac{\text{usage}_{\text{in}}(X)}{U_{\text{in}}}
\end{equation}
ここで:
\begin{itemize}
  \item $\text{usage}_{\text{in}}(X) = |\{t \in \text{scope}(X) : X \in \text{cover}(\text{TCT}, T_t)\}|$は
    スコープ内での使用回数,
  \item $U_{\text{in}} = \sum_{Y \in \text{TCT}} \text{usage}_{\text{in,scope}(X)}(Y)$は
    スコープ$\text{scope}(X)$内での全パターンの総使用回数.
\end{itemize}
\end{definition}

\begin{remark}
KRIMPの符号長（式\ref{eq:code_length}）との相違点は、
分母がデータ全体での総使用回数$U$ではなく、
当該パターンのスコープ内での総使用回数$U_{\text{in}}$となる点である。
スコープ内ではパターンの使用割合が高くなるため、
$\text{usage}_{\text{in}}(X)/U_{\text{in}} > \text{usage}(X)/U$が成り立ち、
より短い符号長が割り当てられる。
\end{remark}

TCT全体の記述長は以下の二部構成をとる:
\begin{equation}\label{eq:tct_total}
  L(\text{TCT}, \mathcal{D}) = L(\text{TCT}) + L(\mathcal{D}|\text{TCT})
\end{equation}

$L(\text{TCT})$（モデルコスト）は各エントリのコストの合計:
\begin{equation}\label{eq:model_cost}
  L(\text{TCT}) = \sum_{(X_i,\mathcal{I}_i,c_i)} \left[
    \underbrace{\sum_{a \in X_i} \ell(a|\text{SCT})}_{\text{パターン符号化}}
    + \underbrace{L_{\mathbb{N}}(|\mathcal{I}_i|) + \sum_{j} \left(L_{\mathbb{N}}(s_{i,j}) + L_{\mathbb{N}}(e_{i,j}-s_{i,j}+1)\right)}_{\text{区間符号化}}
  \right]
\end{equation}
ここで$L_{\mathbb{N}}(\cdot)$はRissanenの汎整数符号~\cite{rissanen1978modeling}、
$\ell(a|\text{SCT})$はStandard Code Tableにおける単体アイテム$a$の符号長である。

$L(\mathcal{D}|\text{TCT})$（データコスト）は各トランザクションの符号化コストの合計:
\begin{equation}\label{eq:data_cost}
  L(\mathcal{D}|\text{TCT}) = \sum_{t=1}^{N} \sum_{X \in \text{cover}(\text{TCT}, T_t)} L_{\text{ts}}(X|\text{TCT})
\end{equation}

\subsection{Dense Interval Compression アルゴリズム}

提案アルゴリズムDense Interval Compression（DIC）は
以下の3段階で構成される:

\begin{enumerate}
  \item \textbf{候補生成}: Aprioriベースの密集区間検出により、
    密集区間を持つアイテムセットを列挙する。
  \item \textbf{Temporal Cover}: 定義\ref{def:tcover}に従い、
    各トランザクションを時間スコープ付きパターンでカバーする。
  \item \textbf{MDL選択}: 貪欲法でTCTにパターンを追加し、
    $L(\text{TCT}) + L(\mathcal{D}|\text{TCT})$が改善する場合にのみ採用する。
\end{enumerate}

\begin{algorithm}[t]
\SetAlgoLined
\KwIn{データベース$\mathcal{D}$, ウィンドウサイズ$W$, 密集閾値$\theta$, 最大パターン長$\ell_{\max}$}
\KwOut{Temporal Code Table $\text{TCT}^*$}

$\text{Candidates} \leftarrow \text{MineDensePatterns}(\mathcal{D}, W, \theta, \ell_{\max})$\;
$\text{TCT}^* \leftarrow \text{SCT}(\mathcal{D})$ \tcp*{Standard Code Table で初期化}
$L^* \leftarrow L(\text{TCT}^*) + L(\mathcal{D}|\text{TCT}^*)$\;

候補をStandard Cover Orderでソート\;

\ForEach{$(X, \mathcal{I}_X) \in \text{Candidates}$}{
  $\text{TCT}' \leftarrow \text{TCT}^* \cup \{(X, \mathcal{I}_X, 0)\}$\;
  全トランザクションに対しTemporal Cover（定義\ref{def:tcover}）を再計算\;
  $L' \leftarrow L(\text{TCT}') + L(\mathcal{D}|\text{TCT}')$\;
  \If{$L' < L^*$}{
    $\text{TCT}^* \leftarrow \text{TCT}'$\;
    $L^* \leftarrow L'$\;
  }
}
\Return{$\text{TCT}^*$}\;
\caption{Dense Interval Compression（DIC）}
\label{alg:dic}
\end{algorithm}

\subsection{理論的保証}

\begin{theorem}[圧縮保証]\label{thm:compression}
アルゴリズム\ref{alg:dic}の出力$\text{TCT}^*$は、
Standard Code Table以上の圧縮を保証する:
\begin{equation}
  L(\text{TCT}^*, \mathcal{D}) \leq L(\text{SCT}, \mathcal{D}).
\end{equation}
\end{theorem}

\begin{proof}
アルゴリズム\ref{alg:dic}はSCTを初期解$\text{TCT}^* = \text{SCT}$として開始する。
各候補$(X, \mathcal{I}_X)$の追加判定（7--10行目）において、
追加後の記述長$L'$が現在の最良値$L^*$を下回る場合にのみ採用する。
したがって、任意のステップで$L(\text{TCT}^*) \leq L^*_{\text{prev}}$が成り立ち、
帰納法により$L(\text{TCT}^*, \mathcal{D}) \leq L(\text{SCT}, \mathcal{D})$が得られる。
\end{proof}

\begin{theorem}[時間的局所性の利得]\label{thm:locality}
パターン$X$がスコープ$\text{scope}(X)$内でのみ出現し（$\text{scope}$外での出現なし）、
スコープサイズ$S_X < N$であるとする。
TCTにおける$X$の符号化利得は:
\begin{equation}\label{eq:locality_gain}
  \Delta L(X) = L_{\text{KRIMP}}(X) - L_{\text{TCT}}(X)
  = \text{usage}(X) \cdot \log_2 \frac{U_{\text{in}}}{U}
  > 0
\end{equation}
ここで$U = \sum_{Y \in \text{CT}} \text{usage}(Y)$は全体での総使用回数、
$U_{\text{in}}$はスコープ内での総使用回数であり、
$U_{\text{in}} < U$（スコープ外に他のパターンの使用が存在する）のとき利得が正となる。
\end{theorem}

\begin{proof}
KRIMPでの$X$に対する合計符号化コスト（全使用を通じた合計）は:
\[
  L_{\text{KRIMP}}(X) = \text{usage}(X) \cdot \left(-\log_2 \frac{\text{usage}(X)}{U}\right).
\]
TCTでの合計符号化コスト（スコープ内使用回数が同一の$\text{usage}(X)$）は:
\[
  L_{\text{TCT}}(X) = \text{usage}(X) \cdot \left(-\log_2 \frac{\text{usage}(X)}{U_{\text{in}}}\right).
\]
差を取ると:
\[
  \Delta L(X) = \text{usage}(X) \cdot \left(-\log_2 \frac{\text{usage}(X)}{U} + \log_2 \frac{\text{usage}(X)}{U_{\text{in}}}\right)
  = \text{usage}(X) \cdot \log_2 \frac{U}{U_{\text{in}}}.
\]
$U_{\text{in}} < U$のとき$\log_2(U/U_{\text{in}}) > 0$であるため、$\Delta L(X) > 0$。
\end{proof}

\begin{remark}
式\ref{eq:locality_gain}の利得は、スコープの狭さ（$S_X \ll N$）に依存する。
直観的には、スコープ外のトランザクションには他のパターンの使用が蓄積するため
$U - U_{\text{in}}$が大きくなり、利得が増大する。
\end{remark}

\subsection{計算量解析}

\begin{proposition}[計算量]\label{prop:complexity}
候補数$C$、トランザクション数$N$、
TCT内のエントリ数$K$（$K \leq C$）、最大パターン長$\ell_{\max}$のとき、
アルゴリズム\ref{alg:dic}の時間計算量は$O(C \cdot N \cdot K \cdot \ell_{\max})$である。
\end{proposition}

\begin{proof}
各候補の追加判定（$C$回のイテレーション）において、
Temporal Coverの再計算が必要である。
1トランザクションのカバーは$K$個のエントリを走査し、
各エントリについてパターンの包含判定に$O(\ell_{\max})$を要する。
$N$個のトランザクションに対してこれを行うため、
1候補あたり$O(N \cdot K \cdot \ell_{\max})$、
全体で$O(C \cdot N \cdot K \cdot \ell_{\max})$となる。
MDL選択により$K \ll C$となるため、実用上の計算量は
候補全採用時よりも大幅に小さい。
\end{proof}

\section{実験}\label{sec:experiments}

本節では、提案手法の有効性を3つの実験で評価する。
実装はPython 3.13で行い、実験環境はApple M1, 16GB RAMである。
再現性のため、ソースコードとデータ生成スクリプトを公開予定である。

\subsection{実験設定}

\textbf{合成データ.}
$N$個のトランザクション、$|\mathcal{I}|$個のアイテムからなる
データベースを生成する。
$P$個の密集パターン（各サイズ$|X|$）を
それぞれ長さ30の区間に埋め込み、
背景ノイズとして各アイテムを独立に確率0.1で出現させる。
シード値を固定し、各条件で5回の独立試行の平均を報告する。

\textbf{ベンチマーク実データ.}
以下の2データセットを使用する:
\begin{itemize}
  \item \textbf{Retail}~\cite{brijs2003retail}: ベルギー小売店の購買記録（88,162トランザクション、16,470アイテム）。
    先頭5,000トランザクションを時系列として使用。
  \item \textbf{Online Retail}~\cite{chen2012online}: 英国ECサイトの購買記録（25,900トランザクション、4,070アイテム）。
    先頭3,000トランザクションを使用。
\end{itemize}

\textbf{ベースライン.}
\begin{itemize}
  \item \textbf{SCT}: Standard Code Table（単体アイテムのみ）。下界参照。
  \item \textbf{KRIMP}~\cite{vreeken2011krimp}: 時間スコープなしのコードテーブル選択。
    提案手法と同一の候補セットを入力として使用。
\end{itemize}

\textbf{パラメータ.}
ウィンドウサイズ$W{=}10$、最小サポート$\theta{=}3$、最大パターン長$\ell_{\max}{=}4$を
デフォルト設定とする。

\textbf{評価指標.}
\begin{itemize}
  \item 圧縮率: $L(\text{TCT}^*)/L(\text{SCT})$（低いほど良い圧縮）
  \item パターン削減率: $1 - |\text{TCT}^*|/|\text{Candidates}|$
  \item KRIMP比: $L(\text{TCT}^*)/L(\text{KRIMP}^*)$（1未満ならTCTが優位）
  \item 実行時間（秒）
\end{itemize}

\section{考察}\label{sec:discussion}

\textbf{時間的局所性の効果.}
実験結果は、密集区間の時間的局所性が
パターンの符号化効率を向上させることを示した（定理\ref{thm:locality}）。
合成データ（E1）では5--14\%の追加圧縮、
実データ（E4）では5--6\%の追加圧縮がKRIMP対比で得られた。
合成データでの改善が大きいのは、
埋め込みパターンが完全に時間局所化されているためであり、
実データではパターンの時間的局所性が不完全であることを反映する。

\textbf{MDLによる自動パラメータ選択.}
ウィンドウサイズ$W$が変化しても選択パターン数が安定する
という結果（E3）は実用上重要である。
従来の密集区間検出では$W$と$\theta$の設定が出力サイズを直接制御するが、
提案手法ではMDL原理が追加的なフィルタリングを行うため、
パラメータ設定への感度が低い。

\textbf{スケーラビリティ.}
現在の実装は各候補追加時にカバー全体を再計算するため、
$O(C \cdot N \cdot K \cdot \ell_{\max})$の計算量を要する。
実データ実験（E4）では$N{=}5000$, $C{=}342$で12.3秒であり、
中規模データには適用可能だが、
$N > 10^5$の大規模データにはSLIMのような差分更新方式の導入が必要である。

\textbf{制約と今後の課題.}
(1) 密集区間の重なりが多い場合のカバー計算の最適化、
(2) 差分更新によるオンライン版TCTの実現、
(3) 密集区間の自動的な粒度選択（階層的な区間構造の導入）が
今後の課題である。


% ===================================================================
\section{おわりに}\label{sec:conclusion}

本研究では、MDL原理と密集区間検出を統合した
パターン要約手法 Temporal Code Table を提案した。
提案手法は以下の特徴を持つ:
(1) 各パターンに時間的有効区間を付与することで
    符号化効率を向上させる（定義\ref{def:tct}）;
(2) Time-Scoped Code Lengthにより時間的局所性を活用し、
    情報理論的に保証された圧縮利得を達成する（定理\ref{thm:locality}）;
(3) 候補パターンの87--93\%を削減しつつ、
    KRIMP対比で5--18ポイント高い圧縮率を達成する。

今後は大規模実データでの評価、
差分更新による計算効率の改善、
およびストリーミング環境への拡張を検討する。
