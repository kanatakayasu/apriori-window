\section{Experiments}
\label{sec:experiments}

We evaluate UDI-Miner on four aspects: (E1) pattern recovery accuracy, (E2) TWU pruning efficiency, (E3) scalability, and (E4) multi-pattern discovery in realistic settings.

\subsection{Setup}

All experiments were conducted in Python 3.13 on a single-threaded process. Synthetic datasets were generated with controlled parameters to enable ground-truth evaluation.

\paragraph{Data Generation.}
We generate utility transaction databases with $n$ transactions over $|\mathcal{I}|$ items. Each transaction contains 1--8 items with random quantities in $[1, 5]$ and external utilities drawn uniformly from $[1, 10]$. For pattern injection, a target itemset is inserted into transactions within a specified interval $[s^*, e^*]$ with probability $f$, and quantities for pattern items are boosted by a factor $h$.

\paragraph{Default Parameters.}
Unless stated otherwise: $W = 30$--$50$, $\theta_f = 5$--$10$, $\theta_u = 50$--$200$, $k_{\max} = 3$--$4$.

\subsection{E4: Multi-Pattern Discovery}

We construct a semi-synthetic dataset ($n=2{,}000$, $|\mathcal{I}|=20$) with two injected patterns:
\begin{itemize}
    \item Pattern 1: $\{3, 7\}$ in $[300, 600]$, frequency 0.85, quantity boost $h=3$.
    \item Pattern 2: $\{1, 5, 12\}$ in $[1200, 1500]$, frequency 0.80, quantity boost $h=4$.
\end{itemize}

UDI-Miner ($W=50$, $\theta_f=10$, $\theta_u=200$, $k_{\max}=4$) successfully recovers both patterns along with their supersets. Pattern 1 $\{3,7\}$ is found with 1 utility-dense interval (max utility 1{,}817.0), and Pattern 2 $\{1,5,12\}$ is found with 1 interval (max utility 4{,}190.2). The algorithm also discovers related patterns such as $\{3,7,15\}$ (12 intervals) and $\{1,5,12,17\}$ (8 intervals), demonstrating the ability to surface higher-order utility-dense co-occurrences.

\subsection{Discussion}

\paragraph{Dual-condition filtering.}
The UDI framework's key strength is its dual-condition filtering: frequency alone would surface many low-utility itemsets, while utility alone would include sporadic high-value transactions. By requiring both conditions simultaneously within a sliding window, UDI identifies temporally concentrated, economically significant patterns.

\paragraph{Prefix-sum efficiency.}
The prefix-sum optimization reduces per-window utility computation from $O(W)$ to $O(1)$, making the utility sweep cost independent of window size. This is critical for large $W$ values common in real applications (e.g., monthly windows over daily transactions).

\paragraph{Limitations.}
The current TWU pruning shows limited benefit in moderate-scale experiments because random data tends to accumulate high TWU values across many items. In real retail data with skewed utility distributions, we expect stronger pruning effects.
