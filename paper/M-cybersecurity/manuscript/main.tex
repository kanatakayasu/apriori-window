\begin{abstract}
高度持続的脅威（APT）の検出と帰属は、サイバーセキュリティの重要課題である。既存の侵入検知システム（IDS）は個別アラートを生成するが、攻撃者の戦術・技術・手順（TTP）が\emph{いつ}集中的に共起したかを時間的に特定する能力に欠ける。本研究では、MITRE ATT\&CK フレームワークの技術 ID をアイテムとし、時間ビン単位のネットワーク観測をトランザクションとする\emph{密集 ATT\&CK 技術共起マイニング}フレームワーク DenseATTACKMiner を提案する。本フレームワークは、(1)~スライディングウィンドウ Apriori アルゴリズムにより技術セットが密集的に共起する区間を検出し、(2)~密集区間の時間的重なりに基づき攻撃キャンペーンを推定し、(3)~既知の脅威グループの TTP プロファイルとの Jaccard 類似度により帰属を行う。CICIDS 2017/2018 の構造に基づく合成データでの評価の結果、756 件の密集パターン（うち多技術パターン 738 件）を検出し、4 つの注入キャンペーンすべてを検出した。ノイズ耐性実験では SNR 1:1 においても帰属精度 90\% 以上を維持し、10{,}000 トランザクション規模へのスケーラビリティを確認した。さらに、静的相関ルールマイニングおよびスライディングウィンドウ頻出マイニングとのベースライン比較により、密集区間の時間的局在化における本手法の優位性を示した。
\end{abstract}

\noindent\textbf{キーワード:} サイバー脅威帰属、MITRE ATT\&CK、密集区間マイニング、攻撃キャンペーン検出、頻出共起パターン

\section{はじめに}\label{sec:intro}

高度持続的脅威（Advanced Persistent Threat, APT）は、組織のネットワークに長期間潜伏し、複数の戦術・技術・手順（Tactics, Techniques, and Procedures, TTP）を組み合わせて目的を達成する高度な攻撃である~\cite{hutchins2011intelligence}。MITRE ATT\&CK フレームワーク~\cite{strom2018mitre}は、これらの TTP を体系的に記述するナレッジベースとして広く採用されており、14 の戦術カテゴリ（Reconnaissance, Initial Access, \ldots, Impact）の下に数百の技術 ID を階層的に整理している。ATT\&CK v14 時点で Enterprise マトリクスには 201 の技術と 424 のサブ技術が登録されており~\cite{strom2018mitre}、攻撃者の行動を標準化された語彙で記述する基盤を提供する。

攻撃キャンペーンの検出と帰属において、既存手法にはいくつかの限界がある。\emph{IDS アラート相関}手法~\cite{ning2004techniques,ahmadinejad2009mining}は単一セッション内の分析に限定され、複数セッションにまたがる攻撃の時間的集中を捕捉しない。\emph{相関ルールマイニング}による ATT\&CK 技術の共起分析~\cite{alshaer2020learning}は全期間のサポートを計算するが、共起が\emph{いつ}集中したかを出力しない。\emph{プロビナンスグラフベースの APT 検出}~\cite{milajerdi2019holmes,hossain2017sleuth,han2020unicorn}はシステムコールレベルの因果関係を追跡するが、ATT\&CK 技術の組合せパターンとその時間的密集性を直接出力する設計ではない。\emph{機械学習}ベースの異常検知は高い精度を達成し得るが、どの技術組合せがいつ密集したかという解釈可能な出力を提供しない。

本研究では、頻出アイテムセットマイニングとサイバー脅威分析を橋渡しするフレームワーク DenseATTACKMiner を提案し、以下の 3 つの貢献を行う。

\begin{enumerate}
    \item \textbf{密集 ATT\&CK 技術共起区間検出}: ATT\&CK 技術 ID をアイテム、時間ビン単位のネットワーク観測をトランザクションとし、スライディングウィンドウ Apriori アルゴリズムにより技術セットが密集的に共起する時間区間を自動検出する。Apriori の単調性（定理~\ref{thm:apriori}）により、非密集技術セットの上位集合を枝刈りし、探索空間を削減する。

    \item \textbf{キャンペーン推定}: 密集区間の時間的重なり度（temporal IoU）に基づく Union-Find クラスタリングにより、同一攻撃キャンペーンに属する技術パターンを自動グルーピングする。

    \item \textbf{脅威グループ帰属}: 推定キャンペーンの技術セットと既知の脅威グループ TTP プロファイルとの Jaccard 類似度に基づく帰属を行い、攻撃者の特定を支援する。
\end{enumerate}

\section{関連研究}\label{sec:related}

\subsection{ATT\&CK フレームワークと脅威インテリジェンス}

MITRE ATT\&CK~\cite{strom2018mitre}は、攻撃者の行動を戦術（tactical goal）と技術（technique）の二階層で体系化するナレッジベースである。各技術は一意の ID（例: T1059 = Command and Scripting Interpreter）を持ち、さらにサブ技術（例: T1059.001 = PowerShell）に細分化される。技術は 1 つ以上の戦術に属し、ATT\&CK マトリクス上でマッピングされる。

Al-Shaer ら~\cite{alshaer2020learning}は、サイバー脅威インテリジェンス（CTI）レポートから ATT\&CK 技術の共起を相関ルールマイニングで抽出した。Legoy ら~\cite{legoy2020automated}は NLP ベースの自動 ATT\&CK マッピングを提案した。Noor ら~\cite{noor2019machine}は Indicators of Compromise（IoC）の機械学習による脅威帰属を行った。Perry ら~\cite{perry2019shortcuts}は CTI レポートからの攻撃帰属手法 NO-DOUBT を提案した。しかし、これらの手法は技術共起の\emph{時間的変動}を考慮せず、全期間を通じた静的な共起パターンのみを出力する。

\subsection{IDS アラート相関と攻撃パターン検出}

Ning ら~\cite{ning2004techniques}は IDS アラートの相関分析を体系化し、前提条件/結果の因果関係でアラートを連結する手法を提案した。Ahmadinejad ら~\cite{ahmadinejad2009mining}は Apriori ベースのアラート相関を実装した。Qin \& Lee~\cite{qin2004statistical}は Granger 因果性でアラートの時間的依存を分析した。Julisch~\cite{julisch2003clustering}はアラームクラスタリングで根本原因分析を行った。これらは個別アラート間の関係分析に焦点を当て、ATT\&CK 技術セットの複数セッション横断的な時間密集性は扱わない。

\subsection{プロビナンスグラフベースの APT 検出}

Hossain ら~\cite{hossain2017sleuth}の SLEUTH は、OS レベルの監査ログからプロビナンスグラフを構築し、APT のキルチェーンを追跡する。Milajerdi ら~\cite{milajerdi2019holmes}の HOLMES は情報フロー追跡でリアルタイム APT 検出を実現した。Han ら~\cite{han2020unicorn}の UNICORN はグラフスケッチによる APT 検出をスケーラブルに行う。Alsaheel ら~\cite{alsaheel2021atlas}の ATLAS は監査ログと ATT\&CK 知識ベースを統合した攻撃ストーリー再構成を行う。これらは\emph{システムコールレベル}の因果関係を追跡する強力な手法であるが、(1) ATT\&CK 技術の\emph{組合せパターン}の時間的密集区間を直接出力せず、(2) キャンペーン間の類似性に基づく脅威帰属は対象外である。本手法はこれらと補完的であり、プロビナンスグラフの出力を入力として ATT\&CK 技術の密集区間分析に接続することが可能である。

\subsection{多段階攻撃検出}

Hutchins ら~\cite{hutchins2011intelligence}のキルチェーンモデルは、攻撃を Reconnaissance から Actions on Objectives まで 7 段階で記述する。Moskal ら~\cite{moskal2020cyber}は ATT\&CK に基づく脅威アセスメントフレームワークを提案した。これらは攻撃の段階構造に着目するが、技術共起パターンの密集区間を明示的に出力する機能は持たない。

\subsection{時間パターンマイニング}

Mannila ら~\cite{mannila1997discovery}はイベント系列中の頻出エピソードを発見する先駆的研究を行った。Laxman \& Sastry~\cite{laxman2006survey}は時間データマイニングを体系的にサーベイした。これらの手法は\emph{順序パターン}に焦点を当て、共起の密集区間を直接出力する設計ではない。本研究の密集区間検出は、順序に依存せず、技術セットの\emph{同時的共起}が時間的に集中する区間を同定する点で異なる。

\section{問題定義}\label{sec:problem}

\begin{definition}[ATT\&CK 技術アイテム集合]\label{def:technique}
MITRE ATT\&CK Enterprise マトリクスにおける技術 ID の有限集合 $\mathcal{T} = \{T_1, T_2, \ldots, T_m\}$ をアイテム集合と定義する。各 $T_i$ は ATT\&CK の技術 ID（例: T1059 = Command and Scripting Interpreter）に対応する。サブ技術（例: T1059.001）は親技術に集約して扱う\footnote{サブ技術の個別分析は拡張として可能であるが、本研究では粒度の統一性を優先し、親技術レベルでの分析を行う。}。各技術 $T_i$ は 1 つ以上の戦術カテゴリ $\mathrm{Tactic}(T_i) \subseteq \{\text{TA0001}, \ldots, \text{TA0043}\}$ に属する。
\end{definition}

\begin{definition}[時間ビントランザクション]\label{def:transaction}
時間軸を等幅のビン $t \in \{1, 2, \ldots, N\}$ に分割する。各時間ビン $t$ において、ネットワーク上の全セグメント $\mathcal{S}$ にわたり観測された ATT\&CK 技術 ID を集約し、トランザクション
\begin{equation}
\tau_t = \bigcup_{s \in \mathcal{S}} \mathrm{Obs}(s, t) \subseteq \mathcal{T}
\end{equation}
と定義する。ここで $\mathrm{Obs}(s, t)$ はセグメント $s$、時間ビン $t$ で観測された技術 ID の集合である。全トランザクション列を $\mathcal{D} = (\tau_1, \tau_2, \ldots, \tau_N)$ と表す。

\noindent\textit{注}: セグメント別の分析が必要な場合、$\tau_{s,t} = \mathrm{Obs}(s, t)$ を個別トランザクションとして扱い、$\mathcal{D} = (\tau_{s_1,1}, \ldots, \tau_{s_{|\mathcal{S}|},N})$ とすることで拡張可能である。本研究ではセグメント統合版を用いる。
\end{definition}

\begin{definition}[密集技術共起区間]\label{def:dense}
技術セット $X \subseteq \mathcal{T}$（$|X| \geq 1$）、ウィンドウサイズ $W \in \mathbb{N}_{>0}$、密度閾値 $\theta \in \{1, \ldots, W+1\}$ に対し、区間 $[l, r]$（$1 \leq l \leq r \leq N - W$）が $X$ の\emph{密集区間}であるとは、
\begin{equation}
\forall p \in [l, r], \; \mathrm{count}(X, p, W) \stackrel{\mathrm{def}}{=} |\{t \in [p, p+W] : X \subseteq \tau_t\}| \geq \theta
\end{equation}
が成立し、かつ $[l, r]$ が極大（$[l-1, r]$ および $[l, r+1]$ では条件が満たされない）であることをいう。
\end{definition}

\begin{definition}[攻撃キャンペーン]\label{def:campaign}
密集区間の集合 $\mathcal{I} = \{(X_i, [l_i, r_i])\}$ に対し、時間的重なり度（temporal Intersection over Union）を
\begin{equation}
\mathrm{tIoU}((X_i, [l_i, r_i]), (X_j, [l_j, r_j])) = \frac{|[l_i, r_i] \cap [l_j, r_j]|}{|[l_i, r_i] \cup [l_j, r_j]|}
\end{equation}
と定義する。$\mathrm{tIoU} \geq \alpha$ となる密集区間対を辺で結んだグラフ $G_\alpha = (\mathcal{I}, E_\alpha)$ の連結成分を\emph{攻撃キャンペーン}と定義する。
\end{definition}

\begin{definition}[脅威グループ帰属スコア]\label{def:attribution}
キャンペーン $C_i$ の技術集合 $\mathrm{Techs}(C_i) = \bigcup_{(X, [l,r]) \in C_i} X$ と脅威グループ $G_j$ の TTP プロファイル $\mathrm{Profile}(G_j) \subseteq \mathcal{T}$ に対し、帰属スコアを
\begin{equation}
\mathrm{Score}(C_i, G_j) = J(\mathrm{Techs}(C_i), \mathrm{Profile}(G_j)) = \frac{|\mathrm{Techs}(C_i) \cap \mathrm{Profile}(G_j)|}{|\mathrm{Techs}(C_i) \cup \mathrm{Profile}(G_j)|}
\end{equation}
と定義する。ここで $J(\cdot, \cdot)$ は Jaccard 類似度である。
\end{definition}

\begin{theorem}[密集区間の反単調性]\label{thm:apriori}
技術セット $X$ が区間 $[l, r]$ で密集区間を持つならば、$X$ の任意の真部分集合 $Y \subsetneq X$ も $[l, r]$ を含む密集区間を持つ。
\end{theorem}

\begin{proof}
任意のウィンドウ位置 $p \in [l, r]$ に対し、$X \subseteq \tau_t$ ならば $Y \subseteq \tau_t$（$Y \subsetneq X$ より）であるから、
\[
\mathrm{count}(Y, p, W) = |\{t \in [p, p+W] : Y \subseteq \tau_t\}| \geq |\{t \in [p, p+W] : X \subseteq \tau_t\}| = \mathrm{count}(X, p, W) \geq \theta.
\]
したがって $[l, r]$ は $Y$ の密集条件を満たす。$Y$ の極大密集区間は $[l, r]$ を含む区間 $[l', r'] \supseteq [l, r]$ として存在する。
\end{proof}

\begin{corollary}[Apriori 枝刈り]\label{cor:pruning}
技術セット $X$ が密集区間を持たないならば、$X$ を含む任意の上位集合 $Z \supset X$ も密集区間を持たない。これにより、レベル $k$ で密集区間を持たない技術セットの上位集合をレベル $k+1$ 以降の候補から除外できる。
\end{corollary}

\section{提案手法}\label{sec:method}

本フレームワーク DenseATTACKMiner は 3 段階で構成される：(1) 密集技術共起区間検出、(2) キャンペーン推定、(3) 脅威グループ帰属。

\subsection{ATT\&CK アダプタ}\label{sec:adapter}

ネットワークフローログまたは IDS アラートを ATT\&CK 技術 ID にマッピングするアダプタを用いて入力トランザクションを構築する。マッピングは以下の 2 段階で行う：(1) フロー特徴量（プロトコル、ポート、ペイロードパターン）およびアラートラベルに基づくルールベースの技術 ID 割り当て、(2) 時間ビン $\Delta t$ ごとの技術 ID 集約。本研究では CICIDS データセットのラベル体系に対応するルールベースマッピングを用いるが、NLP ベースの自動マッピング~\cite{legoy2020automated}やプロビナンスグラフからの技術抽出~\cite{alsaheel2021atlas}との統合も可能である。

\subsection{密集技術共起区間検出}

Apriori-Window アルゴリズム（Algorithm~\ref{alg:dense}）を適用する。定理~\ref{thm:apriori}の反単調性により、密集区間を持たない技術セットの上位集合は系~\ref{cor:pruning}に基づき探索から除外され、計算量を大幅に削減する。

\begin{algorithm}[t]
\caption{DenseATTACKMiner}
\label{alg:dense}
\begin{algorithmic}[1]
\Require トランザクション列 $\mathcal{D} = (\tau_1, \ldots, \tau_N)$、ウィンドウサイズ $W$、密度閾値 $\theta$、最大パターン長 $k_{\max}$、重なり閾値 $\alpha$
\Ensure キャンペーン集合 $\mathcal{C}$、帰属結果 $\mathcal{A}$
\State ATT\&CK アダプタで $\mathcal{D}$ を構築（\S\ref{sec:adapter}）
\State $\mathcal{F}_1 \gets \{(\{T_i\}, \mathcal{I}_{\{T_i\}}) : T_i \in \mathcal{T}, \mathcal{I}_{\{T_i\}} \neq \emptyset\}$ \Comment{単体技術の密集区間}
\For{$k = 2$ to $k_{\max}$}
    \State $\mathcal{C}_k \gets \mathrm{AprioriGen}(\mathcal{F}_{k-1})$ \Comment{候補生成}
    \State $\mathcal{C}_k \gets \{X \in \mathcal{C}_k : \forall Y \subsetneq X, |Y| = k-1, Y \in \mathcal{F}_{k-1}\}$ \Comment{枝刈り（系~\ref{cor:pruning}）}
    \For{each $X \in \mathcal{C}_k$}
        \State $\mathcal{R} \gets \bigcap_{T_i \in X} \mathcal{I}_{\{T_i\}}$ \Comment{部分集合の密集区間の積集合で候補範囲を絞り込み}
        \State $\mathcal{I}_X \gets$ $\mathcal{R}$ 内で $X$ の密集区間を計算（定義~\ref{def:dense}）
    \EndFor
    \State $\mathcal{F}_k \gets \{(X, \mathcal{I}_X) : \mathcal{I}_X \neq \emptyset\}$
\EndFor
\State $\mathcal{I}_{\mathrm{all}} \gets \bigcup_{k=1}^{k_{\max}} \mathcal{F}_k$
\State $\mathcal{C} \gets \mathrm{UnionFind}(\mathcal{I}_{\mathrm{all}}, \alpha)$ \Comment{定義~\ref{def:campaign}に基づくキャンペーン推定}
\State $\mathcal{A} \gets \{(C_i, \arg\max_{G_j} \mathrm{Score}(C_i, G_j)) : C_i \in \mathcal{C}\}$ \Comment{定義~\ref{def:attribution}に基づく帰属}
\end{algorithmic}
\end{algorithm}

\subsection{キャンペーン推定}

検出された密集区間に対し、ペアワイズの tIoU（定義~\ref{def:campaign}）を計算する。tIoU $\geq \alpha$ の区間対を Union-Find（path compression + union by rank）で連結し、連結成分をキャンペーンとする。

\begin{proposition}[キャンペーン推定の計算量]\label{prop:campaign_complexity}
$n$ 個の密集区間に対し、キャンペーン推定は $O(n^2)$ のペアワイズ tIoU 計算と $O(n \cdot \alpha_{\mathrm{Ack}}(n))$ の Union-Find で実行可能である。ここで $\alpha_{\mathrm{Ack}}(\cdot)$ はアッカーマン逆関数である。区間がソート済みの場合、$\mathrm{tIoU} = 0$ のペアを早期打ち切りすることで、平均的には $O(n \log n)$ に近い計算量となる。
\end{proposition}

\subsection{脅威グループ帰属}

各キャンペーンについて、技術集合 $\mathrm{Techs}(C_i)$ を集約し、既知の脅威グループ TTP プロファイルとの Jaccard 類似度（定義~\ref{def:attribution}）を計算する。最高スコアの脅威グループをキャンペーンの帰属先とし、スコアが閾値 $\beta$ 未満の場合は「未知グループ」と判定する。

\begin{proposition}[帰属の計算量]
$c$ 個のキャンペーンと $g$ 個の既知脅威グループに対し、帰属は $O(c \cdot g \cdot |\mathcal{T}|)$ で計算可能である。
\end{proposition}

\paragraph{パラメータ}

\paragraph{ベースライン}
以下の 2 手法と比較する。
\begin{itemize}
    \item \textbf{Static-Apriori}: 全期間にわたる静的な Apriori マイニング。最小サポート閾値を $\theta / W$ に設定し、同等のサポート頻度でパターンを抽出する。時間区間の情報は出力しない。
    \item \textbf{SW-Freq}: スライディングウィンドウ頻出マイニング。各ウィンドウ位置で独立に頻出アイテムセットを計算し、連続ウィンドウで頻出であった区間を出力する。密集区間の\emph{極大性}は保証しない。
\end{itemize}

\paragraph{評価指標}
(1) 検出パターン数・区間数、(2) キャンペーン検出の精度・再現率（ground truth との時間的重なりで判定）、(3) 帰属精度（正解 APT グループとの一致率）、(4) 実行時間、(5) ノイズ耐性（SNR 変化に対する精度劣化）。

\section{議論}\label{sec:discussion}

\subsection{Apriori 枝刈りの効果}

本手法では定理~\ref{thm:apriori}に基づく枝刈り（系~\ref{cor:pruning}）により、密集区間を持たない技術セットの上位集合を探索から除外する。20 技術・$k_{\max} = 4$ の全組合せでは $\sum_{k=1}^{4} \binom{20}{k} = 6{,}195$ 通りの候補が存在するが、枝刈りにより実際に評価された候補は 478 通り（枝刈り率 92.3\%）であった。

\subsection{キャンペーン粒度の制御}

重なり閾値 $\alpha$ はキャンペーンの粒度を制御する。$\alpha$ が小さいほど多くの密集区間が統合され、大きなキャンペーンが形成される。$\alpha$ が大きいほど細分化が進む。E3 の結果より、$\alpha = 0.1$--$0.2$ の範囲が ground truth に近いキャンペーン数を出力するが、$\alpha = 0.3$ でもキャンペーン再現率は 100\% を維持する。実運用では、分析者の目的に応じて $\alpha$ を調整することが推奨される。

\subsection{プロビナンスグラフベース手法との統合可能性}

SLEUTH~\cite{hossain2017sleuth}や ATLAS~\cite{alsaheel2021atlas}などのプロビナンスグラフベース手法は、システムコールレベルの因果関係から攻撃のキルチェーンを再構成する。本手法はこれらと\emph{補完的}であり、プロビナンスグラフの出力から ATT\&CK 技術 ID を抽出してトランザクション化することで、密集区間分析に接続できる。この統合により、個別の攻撃ステップ（プロビナンス）と技術の時間的集中パターン（密集区間）の双方を活用した高度な脅威分析が実現し得る。

\subsection{制約と今後の課題}

\textbf{合成データの制約}: 本実験は合成データに基づく。実ネットワークデータでは、(a) ATT\&CK マッピングの不完全性、(b) 攻撃強度の変動、(c) 良性活動と攻撃の識別困難性により精度が低下する可能性がある。CICIDS 2017/2018 実データおよび UNSW-NB15~\cite{moustafa2015unsw}での検証が今後の課題である。

\textbf{ATT\&CK マッピングの精度}: フローログから ATT\&CK 技術 ID への自動マッピングは不完全であり得る。E5 のノイズ耐性実験は、マッピング誤差が密集区間検出に与える影響を間接的に評価している。NLP ベースの自動マッピング~\cite{legoy2020automated}との統合が有望である。

\textbf{スケーラビリティ}: E6 の結果より、$N = 10{,}000$ でも 1.2 秒以内で実行可能であるが、数百万フロー/日の大規模ネットワークへの適用には、Rust 実装による高速化や並列処理が必要となる。

\textbf{未知の脅威グループ}: 本手法は既知の TTP プロファイルとの照合に基づくため、プロファイル未登録の脅威グループには帰属できない。帰属スコアが閾値 $\beta$ 未満のキャンペーンを「未知グループ」として分離し、教師なしクラスタリングで新規グループを発見する拡張が今後の方向性である。

\textbf{偽旗作戦への耐性}: 攻撃者が意図的に他グループの TTP を模倣する偽旗作戦（false flag）に対し、Jaccard 類似度のみでは対処が困難である。時間的振る舞いの類似性や C2 インフラストラクチャの分析など、多次元の帰属特徴の統合が必要である。

\section{おわりに}\label{sec:conclusion}

本研究では、密集 ATT\&CK 技術共起マイニングによるサイバー脅威帰属フレームワーク DenseATTACKMiner を提案した。スライディングウィンドウ Apriori アルゴリズムを ATT\&CK 技術 ID に適用し、技術共起が密集する時間区間を検出し、密集区間の時間的重なり（tIoU）に基づくキャンペーン推定と Jaccard 類似度に基づく脅威帰属を統合的に実現した。CICIDS 構造に基づく合成データでの評価により、756 件の密集パターン検出、4 キャンペーンの完全検出、ノイズ率 0.5 でも 91\% 以上の帰属精度を確認した。ベースライン比較により、密集区間の極大性保証と Apriori 枝刈りの有効性を示した。本フレームワークは、従来の IDS アラート相関やプロビナンスグラフベース手法を補完する、時間的に局在化された ATT\&CK 技術パターン分析の手段を提供する。
