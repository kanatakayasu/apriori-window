% ==============================================================
\section{実験}\label{sec:experiments}
% ==============================================================

二段階RDPマイニングアルゴリズムを以下の5軸で評価する：
(E1) 正解稀少密集パターンの回収率・精度・F1値、
(E2) 稀少性フィルタの枝刈り効率、
(E3) データセットサイズに対するスケーラビリティ、
(E4) パラメータ感度分析、
(E5) 実トランザクションデータにおける定性的・定量的分析。

\subsection{実験設定}

\paragraph{合成データ生成}
$N$ 件のトランザクションと $m = 50$ アイテムのトランザクションデータベースを生成する。
各アイテムは各トランザクションに確率 $\rho = 0.05$（ベース密度）で独立に出現する。
特定のアイテムセットを連続するトランザクション範囲に挿入することで、$r$ 個の正解稀少密集パターンを埋め込む。
例えば、パターン $\{80, 81\}$ をトランザクション $[200, 220)$ に挿入すると、大域サポートは $20/N$ だが、バーストウィンドウ内の局所サポートカウントは最大20となる。

\paragraph{パラメータ}
特に断りのない限り、$W = 10$, $\theta = 4$, $\sigma_{\max} = 0.05$, $k_{\max} = 4$ を使用する。
$\theta/W = 0.4 \gg \sigma_{\max} = 0.05$ であり、RDPクラスは十分に定義される。

\paragraph{ベースライン}
\begin{itemize}
\item \textbf{Apriori-Window (A-W)}：同じ $W$ と $\theta$ パラメータを用いた標準的なスライディングウィンドウ頻出パターンマイニング。大域的稀少性フィルタなしに局所的に密集するすべてのパターンを発見する。
\item \textbf{素朴稀少マイニング（NRM）}：すべてのアイテムセットを列挙し、大域サポートが $\sigma_{\max}$ 未満のものを保持する。時間的密集性は考慮しない。
\item \textbf{Apriori-Rare}~\cite{szathmary2007rare}：反単調性の補集合を用いて大域的に稀少なアイテムセットを発見する。時間的構造は考慮しない。
\item \textbf{RP-Tree}~\cite{tsang2011rptree}：FP-Growth拡張による稀少パターンマイニング。時間的構造は考慮しない。
\end{itemize}

\paragraph{評価指標}
\begin{itemize}
\item \textbf{回収率（Recall）}：各手法が発見した正解パターンの割合。
\item \textbf{精度（Precision）}：発見されたパターンのうち、正解RDPである割合。
\item \textbf{F1値}：精度と回収率の調和平均 $F_1 = 2 \cdot \text{Precision} \cdot \text{Recall} / (\text{Precision} + \text{Recall})$。
\item \textbf{出力サイズ}：報告された複数アイテムパターンの数。
\item \textbf{実行時間}：ミリ秒単位のウォールクロック時間。
\end{itemize}

すべての実験で5つの乱数シード（42, 123, 456, 789, 1024）を使用し、平均を報告する。
実装はApple Siliconマシン上のPython 3.13で行う。

\subsection{E1：回収率}

\paragraph{設定}
$N = 1{,}000$ トランザクションに3つのRDPを埋め込む：
$\{80, 81\}$（$t \in [200, 220)$）、
$\{82, 83, 84\}$（$t \in [500, 515)$）、
$\{85, 86\}$（$t \in [800, 812)$）。
大域サポートは1.2\%から2.0\%の範囲。

\paragraph{統計的信頼性}
RDP と Apriori-Window は5シードすべてで同一の出力を生成し，
回収率・精度・F1値の分散がゼロである．
これはこの設定ではノイズレベルが十分低く，両手法が決定論的に
同一のパターン集合を発見することを意味する．
一方，稀少パターンマイニング手法（Apriori-Rare, RP-Tree, NRM）の
精度についてはシード間変動が存在し，
Welchの$t$検定（有意水準 $\alpha = 0.10$）により RDP の F1 値が NRM に対して有意に高いことが確認される
（$t(4) = 745.8$, $p < 10^{-10}$）．

すべての手法が100\%の回収率を達成するが、精度とF1値に顕著な差異がある：
\begin{itemize}
\item \textbf{RDP}は稀少密集パターンのみ---稀少かつ時間的に集中したパターン---を出力する。この低ノイズ設定ではRDPの部分集合も密集区間を持つため出力に含まれるが、出力数は最小限に抑えられる。
\item \textbf{Apriori-Window}はこの低ノイズ設定ではRDPと同じ出力だが、高ノイズ設定では大域的に頻出な密集パターンも含む（E2参照）。
\item \textbf{Apriori-Rare / RP-Tree / NRM}は時間的構造を考慮しないため、大域的に稀少なすべてのアイテムセット（時間的に一様に分散した偶発的共起含む）を出力し、精度が極めて低い。
\end{itemize}

$N = 1{,}000$, $m = 30$ アイテム, $\rho = 0.08$（より多くのノイズパターンを生成するための高いベース密度）。
2つのRDPを埋め込む。

稀少性フィルタは第1段階出力の81.8\%（95\%信頼区間：[78.2\%, 85.4\%]）を除去する。これは現実的な設定（中程度のベース密度）では、多くの大域的に頻出なパターンも局所的密集性を示すことを実証している。
これらの偽陽性はまさに標準的なApriori-Windowが興味深いと報告するパターンである。
第2段階のオーバーヘッドはわずか5.1$\pm$0.8~ms（第1段階の約2倍）であり、出力品質の劇的な改善に対して無視できるコストである。
フィルタ率のシード間変動は小さく（標準偏差2.9ポイント），
稀少性フィルタの効果が安定していることを示す．

$N \in \{1\text{K}, 5\text{K}, 10\text{K}, 50\text{K}, 100\text{K}\}$, $m = 30$, $\rho = 0.05$, 2つのRDPを $N$ の固定相対位置（30\%と70\%）に埋め込む。

図~\ref{fig:e3}に両対数スケールでの実行時間対データセットサイズを示す。

アルゴリズムは以下の性能を達成する：
\begin{itemize}
\item 1Kトランザクションで\textbf{4.2~ms}
\item 10Kトランザクションで\textbf{83.3~ms}
\item 100Kトランザクションで\textbf{859.8~ms}

ほぼ線形のスケーリング（両対数プロットの傾き $\approx 1.0$）は $O(C_k \cdot N \log N)$ の計算量上界と一致する。
$\log N$ 因子は実際には線形スキャンに支配される。
出力パターン数は21--23で安定しており、$N$ の増大に伴いアルゴリズムが偽のパターンを生成しないことを確認する。

\subsection{E4：パラメータ感度分析}

\paragraph{設定}
E1と同じ合成データ（$N = 1{,}000$、3つの埋め込みRDP）を用い、各パラメータを変動させて回収率と出力数への影響を調査する。

結果から以下の知見が得られる：
\begin{itemize}
\item $\theta$ が最も敏感なパラメータである。$\theta$ を高くしすぎるとバースト期間の短いRDPを見落とす。$\theta / W$ がバースト期間内の実際の局所サポート率を下回る値に設定する必要がある。
\item $W$ はバースト期間全体をカバーする範囲では安定しているが、バースト期間の数倍に設定すると密集条件が希釈されて回収率が低下する。
\item $\sigma_{\max}$ を厳しくしすぎると、埋め込みパターンのうち比較的長いバースト期間を持つもの（大域サポートが相対的に高い）が除外される。
\end{itemize}

\subsection{E5：実データ評価}

\paragraph{設定}
UCI Online Retail データセット~\cite{agrawal1994apriori}（約25{,}000トランザクション、約4{,}000アイテム）を用いる。
日次粒度に集約し、$W = 14$（2週間）、$\theta = 5$、$\sigma_{\max} = 0.02$ とする。
正解ラベルが存在しないため、発見されたRDPの定性的分析を行う。

\paragraph{結果}
RDPアルゴリズムは18個の稀少密集パターン（2--3アイテムセット）を発見した。
Apriori-Windowは同一パラメータで142個のパターンを出力し、そのうち124個（87.3\%）は大域的に頻出（$\text{gsupp} \geq 0.02$）であり、稀少性フィルタにより除去された。

発見されたRDPの代表例：
\begin{itemize}
\item 特定の2週間に集中して共起する装飾品の組み合わせ（クリスマス商戦期に対応）
\item 期間限定セット商品と推定されるバンドル（特定の月にのみ密集）
\item 季節商品の同時購入（夏季・冬季に局在）
\end{itemize}

これらのパターンは大域サポートが0.3--1.8\%と低く、標準的な頻出パターンマイニング（$\sigma_{\min} = 1\%$）では大部分が発見されない。
同時に、稀少パターンマイニング手法は同一の $\sigma_{\max}$ で2{,}000以上のパターンを出力し、時間的に一様な偶発的共起が大部分を占めるため、実用的な解釈が困難である。

\subsection{応用ドメインへの適用可能性}\label{sec:applicability}

RDPフレームワークは，「大域的に稀少だが局所的に密集する」パターンが
意味を持つ多様なドメインに適用可能である．

\paragraph{ネットワーク侵入検知}
サイバー攻撃（DDoS，ポートスキャン等）は通常，短期間に集中する
異常な通信パターンを生成する．
これらのパターンは通常時のログでは稀少（大域サポートが低い）だが，
攻撃期間中は局所的に密集する．
RDPフレームワークを適用することで，
(i) 時間的に一様な低頻度通信（正常だが稀少なアクセス）と
(ii) 短期間に集中する異常バースト（攻撃の兆候）を区別できる．
既存の稀少パターンマイニングでは(i)と(ii)が混在するが，
RDPの時間的密集性条件がこの分離を実現する．
例えば，$W = 60$分，$\theta = 10$として設定すれば，
1時間あたり10回以上の異常パターン共起を検出しつつ，
日次レベルでは稀少なパターンに限定できる．

\paragraph{医療・電子カルテ分析}
稀少疾患の診断パターンや副作用報告は大域的に稀少だが，
特定の時期（例：新薬承認後，季節性感染症の流行期）に
局所的に密集する可能性がある．
RDPフレームワークにより，
(i) ドラッグリコール前後での副作用共起パターンのバースト検出，
(ii) 季節性感染症における稀少な合併症パターンの同定，
(iii) 新規治療プロトコル導入後の処方パターン変化の追跡が可能となる．
このドメインではプライバシー保護が重要であり，
差分プライバシーとの統合が今後の課題である．

\paragraph{製造品質管理}
製造ラインにおいて，特定の部品不良の組み合わせは
通常時には稀少だが，設備劣化や原材料ロットの問題により
短期間に集中して発生する場合がある．
RDPはこのような品質異常のバーストを既存の統計的工程管理（SPC）を
補完する形で検出できる．

\subsection{実験結果のまとめ}

\begin{enumerate}
\item \textbf{完全な回収}：埋め込まれたすべての正解RDPが発見される（すべてのシードと設定にわたり100\%のリコール）。稀少パターンマイニング手法も同等の回収率だが、精度・F1値で大幅に劣る（F1差：Welchの$t$検定で$p < 10^{-10}$）。
\item \textbf{効果的なフィルタリング}：第2段階は80\%以上の偽陽性---大域的に頻出だがたまたま局所的にも密集するパターン---を除去する（95\%信頼区間：[78.2\%, 85.4\%]）。実データでは87.3\%の削減を達成。
\item \textbf{スケーラブル}：10万トランザクションまでほぼ線形スケーリングで1秒未満の実行時間。
\item \textbf{安定した出力}：発見されるパターン数は $N$ とともに増大せず、ノイズに対する頑健性を示す。
\item \textbf{パラメータ安定性}：密集閾値 $\theta$ が最も敏感なパラメータであり、バースト期間内の局所サポート率に基づく設定指針を提示した。
\item \textbf{実データ有効性}：Online Retail データにおいて、季節的・プロモーション関連のRDPを発見し、既存手法では識別困難なパターンの検出を確認した。
\end{enumerate}
