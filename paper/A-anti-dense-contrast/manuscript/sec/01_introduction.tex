% ==============================================================
\section{はじめに}\label{sec:intro}
% ==============================================================

頻出パターンマイニングは，Aprioriアルゴリズム~\cite{agrawal1994apriori}の提案以来，
データマイニング研究の基盤となってきた．
トランザクションデータベースにおいて\emph{頻出}するパターンの発見に関しては
膨大な研究がなされてきたが，パターン出現の\emph{時間的構造}---すなわち
パターンが\emph{いつ}頻出し\emph{いつ}頻出しないか---に対する関心は
相対的に乏しかった．

密集区間マイニング~\cite{dong2012contrast}は，パターンのスライディングウィンドウ
サポートが所定の閾値を超える連続的な時間区間を特定することで，この空白を埋める．
これらの区間は持続的な高活性期間を捉え，イベント帰属や購買行動の時間分析において
有用性が実証されている．
しかし，既存のフレームワークはコインの片面のみを扱っている．
すなわち，\emph{存在}（高サポート）は特徴づけるが，
\emph{不在}（低サポート）を対称的に特徴づける機構が欠けている．

この非対称性は，密集区間マイニングの分析力をいくつかの重要な点で制限する．
第一に，以前人気であったパターンが\emph{消失}する期間---例えば販促キャンペーンの
終了など，ビジネス上重要な現象---を直接特定できない．
第二に，異なるレジーム間（例：介入前後）でパターンの時間的構造が
どのように変化するかを比較する原理的枠組みが存在しない．
第三に，形式的な「反密集」概念の不在は，低頻度時間パターンのマイニングを
加速しうる反単調性に類似した枝刈り戦略の適用を妨げている．
なお，密集区間の補集合を取ることで低サポート期間を近似的に特定することは
可能であるが，この素朴なアプローチには二つの問題がある．
第一に，高閾値$\theta$と低閾値$\theta_{\text{low}}$の間の
中間帯域が無視されるため，サポートが緩やかに低下する過渡期を
反密集と誤認する．
第二に，密集区間の反単調性（上位集合は部分集合の密集区間を含まない）とは
\emph{逆方向}の単調性が反密集区間では成立し（定理~\ref{thm:anti_monotone}），
これを利用したトップダウン枝刈りは補集合操作では得られない独自の探索戦略を可能にする．

本論文では，密集区間マイニングフレームワークの二つの対称的拡張を提案する：

\begin{enumerate}
\item \textbf{反密集区間}：密集区間の双対概念として，パターンのサポートが
低閾値$\theta_{\text{low}}$を\emph{下回り}続ける極大連続区間を定義する．
反密集区間は持続的な不在を捉え，パターン消失期間の直接的な特定を可能にする．

\item \textbf{コントラスト密集パターン}：二つの時間レジーム間で
パターンの密集区間\emph{構造}を比較するフレームワークを導入する．
四種類の正準トポロジー変化（出現・消失・増幅・収縮）の分類体系を定義し，
観測された構造変化の有意性を評価するための置換検定を提供する．
\end{enumerate}

本論文の貢献は以下のとおりである：
\begin{itemize}
\item 反密集区間を形式化し，効率的なトップダウン探索を可能にする
逆単調性を証明する（第~\ref{sec:problem}節）．
\item パターンのトポロジー変化の完全な分類体系と統計的有意性のための
置換検定を備えたコントラスト密集パターンフレームワークを定義する（第~\ref{sec:problem}節）．
\item 反密集区間とコントラストパターンを既存の密集区間検出と同じ
計算量で算出する効率的なアルゴリズムを提示する（第~\ref{sec:method}節）．
\item 既知のグラウンドトゥルースを持つ合成データおよびシミュレートされた
小売シナリオにより提案手法を検証し，反密集区間の効果的な復元（F1 = 0.789），
トポロジー変化の正確な分類（76\%の精度），およびキャンペーン誘発パターン消失の
統計的に有意な検出を実証する（第~\ref{sec:experiments}節）．
\end{itemize}

本論文の構成は以下のとおりである．
第~\ref{sec:related}節で関連研究をレビューする．
第~\ref{sec:problem}節で形式的な問題定義を示す．
第~\ref{sec:method}節で提案アルゴリズムを記述する．
第~\ref{sec:experiments}節で実験結果を報告する．
第~\ref{sec:conclusion}節で今後の方向性とともに結論を述べる．
