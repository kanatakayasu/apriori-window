# 多次元密集領域マイニング

**新規性**: Very High | **難度**: 4/5 | **投稿先**: KDD / ICDM / TKDE

## 概要

現在の Apriori-Window は 1D 時間軸上のスライディングウィンドウで密集区間を検出する。
これを 2D 以上の多次元トランザクションデータに拡張し、
パターンの「密集領域」を多次元空間上のレベルセットとして定式化する。

## 核心的新概念: サポート曲面と密集領域

```
現在:  s_P(l) → ℝ          (1D関数, 出力は1D区間 [s, e])
拡張:  s_P(a₁, a₂, …, aₙ) → ℝ  (nD関数, 出力はnD密集領域 R_P)
```

### 定式化

```
Definition (Multi-Dimensional Dense Region):
  トランザクションDB D の各トランザクション i は座標 c_i = (c_i^1, ..., c_i^N) ∈ ℝ^N を持つ。
  窓サイズベクトル W = (W_1, ..., W_N)、サポート閾値 θ に対し、

  R_P = {a ∈ ℝ^N : |{i : c_i ∈ Box(a, W) ∧ P ⊆ I_i}| ≥ θ}

  Box(a, W) = [a^1, a^1+W_1] × ... × [a^N, a^N+W_N]
```

## 理論的性質

### 1. 密集領域包含定理

P ⊂ Q ならば R_Q ⊆ R_P。

スーパーセットの密集領域はサブセットの密集領域に必ず含まれる。
1D では区間包含だが、nD では領域包含に一般化される。
→ Apriori 的枝刈りが多次元でもそのまま適用可能。

### 2. 次元分解可能性

独立な次元間では、nD 密集領域は各次元の 1D 密集区間の交差で計算可能:

```
R_P ≈ R_P^{dim1} ∩ R_P^{dim2} ∩ ... ∩ R_P^{dimN}
```

→ 計算量が O(N^d) から O(d·N) に削減される（独立性が成立する場合）。

### 3. サポート曲面の区分定数性

サポート曲面 s_P(a) は高々 O(N^d) 個の異なる値を持つ区分定数関数。
各トランザクションの出入りが変化を引き起こす。

## 具体的定式化パターン

### A. 時間 × 空間 (2D連続)

- 各トランザクション: 座標 (t, x) — 時刻と空間位置
- 窓: 矩形 [t-W_t/2, t+W_t/2] × [x-W_x/2, x+W_x/2]
- 出力: 2D 平面上の連結密集領域
- 窓形状: 矩形（分離可能、高速）/ 円形（等方的）/ 適応的

### B. 時間 × カテゴリ (2D離散)

- 各トランザクション: (t, g) — 時刻と離散グループ（店舗支店、ユーザセグメント等）
- 出力: (パターン, 時間区間, グループ集合) の3つ組
- グループ間の「近傍」定義が必要（地理的隣接等）

### C. 一般 N 次元テンソル

- 連続・離散次元の任意の組み合わせ
- 出力: N 次元連結成分としての密集領域

## 既存研究との差異

| 既存手法 | 類似点 | 本拡張との違い |
|---------|--------|--------------|
| DBSCAN / OPTICS | 多次元空間の密集検出 | 点のクラスタであり、パターン（アイテムセット）ごとの密集領域ではない |
| 空間的共局在マイニング | 特徴の空間的共起 | グローバルサポート、スライディングウィンドウなし |
| DenseAlert (KDD 2017) | ストリーミングテンソルの密部分検出 | アイテムセット意味論なし、生の計数密度のみ |
| CatchCore / M-Zoom | 階層的密部分テンソル | 静的テンソル、窓なし、アイテムセット解釈なし |
| 時空間頻出系列パターン | 空間注釈付き系列マイニング | 系列であってアイテムセットではない、密集区間概念なし |
| エマージングパターン | サポート変化の検出 | 2 つの静的データセット比較であり、連続多次元領域ではない |

**ギャップ**: 多次元トランザクション空間上のスライディングウィンドウで局所的サポート曲面を計算し、そのレベルセットとして密集領域を定義する手法は存在しない。

## アルゴリズム的課題と解決策

### 計算量爆発

| 次元 | アンカー点数 | 各点のサポート計算 |
|------|-----------|-----------------|
| 1D | O(N) | O(log N) |
| 2D | O(N²) worst case | O(log² N) (2D range tree) |
| nD | O(N^d) worst case | O(log^d N) |

### 解決策

1. **グリッド離散化**: 連続次元をビンに分割。100×100 グリッド = 10,000 点（実用的）
2. **疎評価**: トランザクションが存在するアンカー点のみ評価
3. **次元分解**: 各次元独立に 1D 密集区間を計算 → 交差。O(N^d) → O(d·N)
4. **スウィープ面法**: 2D で一軸に沿ってスウィープし、他軸の 1D 窓構造を維持。O(N log N)

## 応用シナリオ

### 小売分析 (時間 × 地理)

- データ: チェーン店舗の POS データ（時刻 + 店舗位置）
- 問い: 「{オーガニック牛乳, アボカド} がいつ・どの地域で密集しているか？」
- 窓: W_time=1週間, W_space=50km
- 価値: 全国展開前の地域限定トレンド早期検出

### 疫学 (時間 × 緯度 × 経度)

- データ: 診断記録（日付 + 患者所在地）
- 問い: 「{発熱, 咳, 倦怠感} の共起がどこで・いつ背景率を超えるか？」
- 窓: W_time=2週間, W_lat=0.5度, W_lon=0.5度
- 価値: SaTScan と異なり症状パターンの密度を検出（新興感染症の特徴的症状シグネチャ）

### IoT センサーネットワーク (時間 × センサーグリッド)

- データ: グリッド配置デバイスのセンサーイベント
- 問い: 「{高温, 低湿度, 高CO2} がどの空間-時間ゾーンで集中しているか？」
- 窓: W_time=1時間, W_x=10センサー, W_y=10センサー

### ソーシャルメディア / EC (時間 × ユーザ属性)

- データ: ユーザ行動（時刻 + 年齢層・所得層等）
- 問い: 「{ゲーム機, ストリーミング契約} がどのデモグラフィック×時期で密集？」

## 研究ロードマップ

| Phase | 内容 | 期間 |
|-------|------|------|
| A. 定式化 | 問題定義、密集領域包含定理の証明、計算量解析 | 2-3週間 |
| B. 2Dプロトタイプ | 時間×空間を Python で実装、合成データで検証 | 2-3週間 |
| C. 効率的アルゴリズム | スウィープ面法の開発、計算量証明、ナイーブとの比較 | 3-4週間 |
| D. 実データ検証 | Dunnhumby（店舗位置付き）or 疫学データセット | 2-3週間 |
| E. 論文執筆 | KDD or ICDM ターゲット | 3-4週間 |

## リスク評価

| リスク | 深刻度 | 緩和策 |
|--------|--------|--------|
| 計算コストが実データに対して過大 | 中 | グリッド離散化、次元分解、GPU加速 |
| 密集領域が自明（全空間 or 空集合） | 低 | 適切な窓サイズ選択、合成データで事前検証 |
| 「グリッド上の 1D Apriori-Window」と見なされる | 中 | 理論的性質（包含定理、分解可能性）と次元横断パターンの実証で差別化 |
