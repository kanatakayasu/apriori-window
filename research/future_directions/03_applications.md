# 応用ドメイン駆動の拡張

Apriori-Window フレームワーク（密集区間検出 + イベント帰属パイプライン）の
ドメイン応用によるインパクト創出。

---

## Apriori-Window の汎用的強み

1. **密集区間検出**: パターンが「いつ」局所的に頻出かを同定（グローバル頻度ではなく）
2. **イベント帰属**: サポート変動を外部イベントに統計的に結びつける

この組み合わせは多くのドメインで未適用。

---

## 1. 薬剤疫学

**推奨投稿先**: JAMIA / AMIA / Drug Safety | **データ入手性**: 高

### マッピング

| 概念 | 本ドメインでの対応 |
|------|------------------|
| アイテム | 処方薬（ATC コード） |
| トランザクション | 患者の受診記録 |
| バスケット | 同一受診内の複数処方 |
| 窓 | カレンダー月のスライディングウィンドウ |
| イベント | FDA 警告、診療ガイドライン改訂、保険償還変更 |

### 問い
「オピオイド + ベンゾジアゼピンの共処方パターンが異常に密集/疎になった時期はいつで、
それはどの規制イベントに帰属するか？」

### 既存手法の限界
- 不均衡分析 (PRR, ROR): 静的なシグナル強度であり、時間ダイナミクスなし
- CausalImpact: 単一時系列の介入効果、組み合わせパターンのサポートには非対応
- エマージングパターン: 2 つの静的 DB 比較、連続的時間発展なし

### 利用可能データ
- MIMIC-IV (処方+診断+時刻)
- Medicare/Medicaid Claims (大規模、申請要)

## 2. 製造業故障診断

**推奨投稿先**: IEEE Trans. Semiconductor Manufacturing / KDD Applied | **データ入手性**: 中

### マッピング

| 概念 | 本ドメインでの対応 |
|------|------------------|
| アイテム | センサーアラームタイプ / 故障コード |
| トランザクション | 時間ビン（1分/5分）ごとのアラーム集約 |
| バスケット | 同一装置ユニットからのアラーム |
| 窓 | 数時間〜数日のスライディングウィンドウ |
| イベント | 予防保全、部品交換、原材料ロット変更、レシピ変更 |

### 問い
「{温度高, 振動過大, 圧力低} のアラーム共起が異常に密集した時期はいつで、
それはどの保全イベントに帰属するか？」

### 既存手法の限界
- FDD (PCA, 深層学習): 「何かがおかしい」は検出するが、どのアラームパターンがなぜ増えたかは説明しない
- 因果 AI: 明示的変数モデリングが必要、組み合わせ的アラームパターンには不向き

### 利用可能データ
- SECOM データセット（UCI、半導体製造）
- 産業連携が望ましい

## 3. サイバーセキュリティ

**推奨投稿先**: USENIX Security / RAID / ACSAC | **データ入手性**: 中

### マッピング

| 概念 | 本ドメインでの対応 |
|------|------------------|
| アイテム | アラートタイプ / MITRE ATT&CK 技術 ID |
| トランザクション | 時間ビン（1分）ごとのネットワークセグメント別アラート |
| バスケット | 同一 src/dst IP ペアからのアラート |
| 窓 | 数分〜数時間 |
| イベント | パッチ適用、ファイアウォールルール変更、新サービス公開、CVE 公開 |

### 問い
「火曜のパッチ適用は、実際にラテラルムーブメントパターンを減少させたか？」

### 既存手法の限界
- ChronoCTI (2024): CTI レポート（テキスト）からのマイニング、ライブアラートストリームではない
- ADROIT: 時空間相関攻撃段階の検出はするがイベント帰属なし

### 利用可能データ
- CICIDS 2017/2018
- DARPA データセット

## 4. 臨床ゲノミクス

**推奨投稿先**: Bioinformatics / ISMB / Genome Research | **データ入手性**: 高

### マッピング

| 概念 | 本ドメインでの対応 |
|------|------------------|
| アイテム | 発現閾値以上の遺伝子 |
| トランザクション | 個々の細胞（擬時間で順序付け） |
| 窓 | 擬時間軸上のスライディングウィンドウ |
| イベント | 発達転換点（原腸陥入、神経管形成）、薬剤処理、CRISPR ノックアウト |

### 問い
「遺伝子 X, Y, Z の共発現は原腸陥入期の狭い擬時間窓でのみ密集し、
遺伝子 K のノックアウトによりこのバーストが消失するか？」

### 既存手法の限界
- WGCNA: 静的な共発現ネットワーク、「いつ」モジュールが密集するかは追跡しない
- TIME-CoExpress (2025): ペアワイズのみ、高次アイテムセットなし、イベント帰属なし
- 差次的共発現解析: 2 条件の静的比較

### 利用可能データ
- 公開 scRNA-seq データセット多数（GEO, Human Cell Atlas）

## 応用論文の共通構成

1. **問題定義**: ドメイン固有のマッピングを明確化
2. **アダプタ実装**: ドメインデータ → Apriori-Window の入力形式への変換
3. **公開データでの概念実証**: 既知のイベントに対する帰属結果の検証
4. **ドメイン専門家による定性評価**: 検出されたパターン-イベント帰属の妥当性
5. **手法論文の引用**: Apriori-Window 論文をアルゴリズム基盤として引用
