# Event Attribution Pipeline — 実験レポート（最終版）

> 実施日: 2026-03-17
> ブランチ: feature/direction-e-event-attribution
> 改訂: 3回目（アイテム重複排除の導入後）

---

## 1. 実験目的

Event Attribution Pipeline（Phase 2）の性能を以下の4軸で検証する:

| 実験 | 問い |
|------|------|
| E1 | 既知の帰属関係を正しく検出できるか（合成データ・正解率検証） |
| E2 | パラメータ変化に対する感度はどうか |
| E3 | データサイズ・イベント数に対するスケーラビリティ |
| E4 | 実データ（公開ベンチマーク）に合成イベントを注入した場合の挙動 |

---

## 2. 実装の反復改善

3段階の改善を経て現在の実装に到達した。

### 第1版（初回実験）

基本パイプライン: パターン単位の Bonferroni 補正。

**結果**: E1a F1=0.05, Recall=0.27。高い FP 率とパターン単位の多重検定未制御が原因。

### 第2版（改善1: グローバル BH 補正 + フィルタ）

| 改善 | 内容 |
|------|------|
| グローバル BH 補正 | 全パターン × 全イベントの仮説を一括で BH (FDR) 補正 |
| Level-shift magnitude | 変化点の magnitude を窓平均の差に変更 |
| min_support_range フィルタ | サポート振幅の小さいパターンを除外（仮説数 310 → 23） |

**結果**: E1a Recall=1.00, Precision=0.15, F1=0.26。Recall は改善したが FP 率は依然高い。

### 第3版（改善2: アイテム重複パターン排除）— 現在の実装

**根本原因分析**: FP の 100% が「副次パターン」（植え込みアイテムを1つ含む非 GT パターン）。例: 植え込みパターン `(1001, 1002)` に対して `(78, 1001)` が FP。アイテム 1001 がブーストされるため、1001 を含む全パターンが連動。

**解決策**: 同一イベントに帰属されたパターンのうち、アイテムが重複するものを Union-Find でクラスタリングし、各クラスタから最高スコアのパターンのみを残す。

**結果**: E1a F1=1.00（全5 seed で完璧）。

| 指標 | 第1版 | 第2版 | 第3版 |
|------|-------|-------|-------|
| E1a Recall | 0.27 | 1.00 | **1.00** |
| E1a Precision | 0.03 | 0.15 | **1.00** |
| E1a F1 | 0.05 | 0.26 | **1.00** |

## 3. 実験設計

### 合成データ生成

- ベース語彙: 200 アイテム、各アイテムの出現確率 p\_base = 0.03
- 植え込みパターン: 語彙外アイテム (1001+) を使用。イベント期間中に全パターンアイテムを同時に確率 boost\_factor で挿入
- デコイイベント: パターンとは無関係な時間帯に配置
- Ground Truth: (植え込みパターン, イベントID) のペア

### 評価指標

- **厳密一致**: 予測パターンが GT パターンと完全一致
- Precision = TP / (TP + FP), Recall = TP / (TP + FN), F1

### パイプライン設定（デフォルト）

| パラメータ | 値 |
|-----------|-----|
| window\_size | 50 |
| min\_support | 3 |
| max\_length | 2 |
| change\_method | threshold\_crossing |
| sigma | window\_size (50) |
| n\_permutations | 5000 |
| alpha | 0.20 |
| correction\_method | bh |
| global\_correction | True |
| min\_support\_range | 5 |
| deduplicate\_overlap | True |

## 5. 考察

### 5.1 アイテム重複排除の効果

本改善の核心であるアイテム重複排除は、**副次パターン問題を完全に解決**した。

| 実験 | FP (排除前) | FP (排除後) | 削減率 |
|------|-----------|-----------|--------|
| E1a seed=0 | 20 | 0 | 100% |
| E1c seed=0 | 103 | 1 | 99% |
| E4 T10I4D100K | 319 | 13 | 96% |
| E4 retail | 258 | 5 | 98% |

この手法が有効な理由:
- 植え込みアイテム X がブーストされると、X を含む全パターン `{X, Y}` のサポートが連動変動
- Union-Find によるクラスタリングでこれらを同一グループに統合
- 最高スコアのパターン（通常は植え込みパターン自体）のみ残す

### 5.2 パラメータ・ロバスト性の向上

deduplication 導入前後で、パラメータ感度が大幅に低下:

| パラメータ | 改善前 | 改善後 |
|-----------|--------|--------|
| window\_size | F1 が 0-1.0 で大変動 | 10-50 で全て F1=1.0 |
| sigma | F1 が 0.24-0.75 | 全値で F1=1.0 |
| n\_permutations | F1 ≈ 0.24 で一定 | 全値で F1=1.0 |

deduplication がパターン選択の主力となり、統計検定パラメータの影響が相対的に小さくなった。

### 5.3 残る制約

#### 1. イベント時間窓の重複

E1c の一貫した 1 FP（`(1003, 1004) → Decoy_1`）は、デコイイベントの時間窓が植え込みイベントと重複している場合に発生する。これは**統計的に正当な検出**であり、アルゴリズムの限界ではなく評価指標の限界である。

#### 2. 高ベースラインサポートへの注入

E4 retail で Recall=0 の原因は、注入先が元々高頻度のアイテムペアであること。小売データでは人気商品ペアのサポートが自然に大きく変動するため、ブーストの相対効果が薄い。**語彙外アイテムの注入**（合成データ方式）であれば検出可能と推測される。

#### 3. 密集イベント

E3 n\_events=20 で F1=0.05 となったのは、5000 トランザクションに 20 イベントが詰め込まれ、イベント間隔 (~238) がイベント幅 (200) とほぼ同一であるため。この条件では円形シフト検定の帰無分布が偏り、検定の弁別力が低下する。

#### 4. alpha の下限

BH 補正後の調整済み p 値が 0.05-0.12 程度であるため、従来的な alpha=0.05 では多くの正当な帰属が棄却される。これは置換検定の分解能（n\_permutations の上限）と仮説数に起因する構造的制約。

### 5.4 スケーラビリティ

| フェーズ | 計算量 | ボトルネック度 |
|---------|--------|-------------|
| Phase 1 (Apriori) | O(N × \|I\|²) | 高（~50%） |
| Support 時系列 | O(N × \|P\|) | 中（~40%） |
| Attribution | O(\|P\| × \|E\| × n\_perm) | 低（~10%） |
| Deduplication | O(\|R\| × max\_items) | 無視可能 |

N=100K でも 70 秒と実用的。deduplication のオーバーヘッドは O(significant results × max items per pattern) で無視可能。

---

## 6. 結論

3段階の反復改善により、Event Attribution Pipeline は合成データで**完璧な検出性能**（F1=1.0）を達成した:

1. **グローバル BH 補正**: パターン間の多重検定を適切に制御
2. **min\_support\_range フィルタ**: 仮説空間を 90% 以上削減
3. **アイテム重複排除**: 副次パターンを完全除去（**最も効果的な改善**）

**主要な成果**:
- E1a-E1b: 全条件で F1=1.0
- E1c (重複イベント): F1=0.86（残り FP は統計的に正当な検出）
- E1d: デコイ0 で F1=1.0、デコイ増加でも 0.51-1.00
- E4 T10I4D100K: FP を 96% 削減（319 → 13）、Recall=1.0 維持
- パラメータ・ロバスト性: window\_size=10-50, sigma 全値, n\_permutations=50-1000 で F1=1.0

**残る課題**: 高ベースラインサポートへの注入検出（E4 retail）と、密集イベント環境（E3 n\_events=20）は今後の研究課題として残る。
