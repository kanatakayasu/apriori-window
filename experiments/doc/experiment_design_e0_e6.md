## E0: 厳密性検証

### 目的・示唆

> 提案手法は定義に対して厳密である（FP=0, FN=0）

E1〜E6 の比較実験の前提条件を保証する。E0 が成立しない限り後続比較は全て無意味になる。査読者への「実装バグがないこと」の証明でもある。

### 設計

| 項目 | 内容 |
|------|------|
| データ | 小規模合成データ（N=10〜100, アイテム数 10〜20） |
| Oracle | 全アイテムセット・全区間を総当たりで列挙 |
| 比較 | 提案法の出力 == Oracle の出力 |
| 指標 | precision=1, recall=1（全パラメータ条件） |
| sweep | min_support / window_size / λ_baskets の代表値を組み合わせ |

### 実装方針

既存の `cargo test` / `pytest` に小規模 Oracle テストとして追加する。論文本文には「全条件で完全一致を確認した」と一文記載し、詳細は補遺に集約する。

## E1: 問題設定ミスマッチの可視化

### 目的・示唆

> flatten は構造的に偽共起を生む。その深刻さはバスケット密度に比例する

「なぜ提案手法が必要か」を定量的に示す実験。flatten の失敗がデータの性質（バスケット密度）に依存することを示し、問題の構造的原因を定量化する。

### 設計

| 項目 | 内容 |
|------|------|
| sweep | λ_baskets ∈ {1.0, 1.5, 2.0, 3.0, 5.0} |
| 固定 | N=10,000, G=10, p_same=0.8, min_support=50, window_size=500, max_length=4 |
| seeds | 0..4（5本） |

### 比較手法

| 区分 | 手法 |
|------|------|
| 主比較 | Phase 1, 従来 Apriori-window（flatten） |
| 参考（異目的） | Apriori, FP-Growth, Eclat, LCM |

参考群は全体支持度ベースであり密集区間を出力しない。パターン集合の傾向比較のみ行い、速度比較は主張しない。

### 評価指標

| 指標 | 定義 |
|------|------|
| GTSPR | 偽共起パターン率（Ground Truth SPR） |
| Observed Spurious | 従来法のみ検出した多項目パターン数 |
| Multi-item Pattern Count | 検出パターン数（len≥2） |
| 実行時間 | wall-clock（ms） |

### 期待される結論

- λ_baskets 増加で従来法の GTSPR が単調増加、提案法は GTSPR=0 を維持
- Observed Spurious が急増（参考値: λ=1.0 で約 1 → λ=5.0 で約 379）

## E2: 構造感度マップ

### 目的・示唆

> 偽共起リスクはデータの構造（カテゴリ粒度・同カテゴリ偏り）によって系統的に変わる

E1 が「バスケット数の増加」という量的な軸を示したのに対し、E2 は「データの質的構造」という軸を示す。実務者が自分のデータのリスクを事前評価できる根拠になる。

### 設計

| 項目 | 内容 |
|------|------|
| sweep 軸 A | G（カテゴリ数）∈ {3, 5, 10, 20, 50} |
| sweep 軸 B | p_same ∈ {0.5, 0.6, 0.7, 0.8, 0.9}（可能なら2軸化） |
| 固定 | N=10,000, λ_baskets=2.0, 他は E1 準拠 |
| seeds | 0..4（5本） |
| 比較手法 | Phase 1, 従来 Apriori-window（flatten） |

### 評価指標

GTSPR, Observed Spurious（E1 と同一指標）

### 期待される結論

- G が小さいほど GTSPR 高（カテゴリが粗いと偶発的共起が多い）
- p_same が高いほど GTSPR 高（同カテゴリ集中により偽共起が増加）

## E3: スケーラビリティ

### 目的・示唆

> 提案手法は厳密性を保ったまま実用規模で動作する。flatten よりスケールが良い

「正しいが遅すぎて使えない」という懸念への反証。flatten は候補爆発でスケールが悪化するが、提案法はバスケット粒度の枝刈りが効いて相対的に有利になることを示す。

### 設計

| 項目 | 内容 |
|------|------|
| sweep | N_transactions ∈ {10^3, 10^4, 10^5, 10^6} |
| 固定 | λ_baskets=2.0, G=10, 他は E1 準拠 |
| seeds | 0..2（3本） |
| 比較手法 | Phase 1, 従来 Apriori-window（flatten） |

### 評価指標

| 指標 | 定義 |
|------|------|
| 実行時間 | wall-clock（ms） |
| peak RSS | 最大常駐メモリ（MB） |
| Trad./P1 比 | 従来法 / Phase 1 の時間比 |

### 追加 sweep（可能なら）

min_support と max_length を振って計算コストの境界を明示する。

## E5: 区間検出評価

### 目的・示唆

> flatten の問題はパターン集合の誤りにとどまらず、密集区間の時間的境界も歪める

E1 との対比で「パターンが偽物なら、その区間も汚染される」を区間レベルで定量化する。E1 が「何を検出するか」の誤りを示すのに対し、E5 は「いつ密集するか」の誤りを示す。

### 前提確認

この実験が E1 に対して独自の示唆を持つ条件:

> 真の共起パターン（GTSPR には含まれない）でも、flatten による支持度の底上げで区間境界が膨張することを確認できた場合

実施前に既存 CSV で区間膨張の有無を確認する。膨張が見られない場合は本実験の位置づけを「E1 の区間レベル補強」に格下げする。

### 設計

| 項目 | 内容 |
|------|------|
| sweep | λ_baskets ∈ {1.0, 1.5, 2.0, 3.0, 5.0}（E1 と同一条件） |
| 固定 | E1 と同一 |
| seeds | 0..4（5本） |

### 比較手法

| 区分 | 手法 |
|------|------|
| 主比較 | Phase 1, 従来 Apriori-window（flatten） |
| 参考 | LPFIM, LPPM（異なる問題定義。出力傾向の参考のみ、速度比較なし） |

### GT 定義

| GT の種類 | 定義 |
|-----------|------|
| 真の密集区間 | true_support(P, I) ≥ θ かつ \|I\| ≥ W を満たす区間 |
| 汚染区間 | txn_support(P, I) ≥ θ かつ true_support(P, I) < θ の区間 |

### 評価指標

| 指標 | 定義 |
|------|------|
| Interval Purity | 検出区間内でバスケット内共起が成立するトランザクションの割合の平均 |
| Mean Jaccard | GT 区間との重複度（パターン平均） |
| Spurious Interval Rate | flatten のみ検出した区間 / 全検出区間 |

### 期待される結論

- λ_baskets 増加で flatten の Interval Purity が低下、Phase 1 は高水準を維持
- Mean Jaccard は Phase 1 > flatten（E1 のパターン結果と対応）
- 非偽共起パターンでも区間膨張が発生すれば E1 への付加価値あり

## E4: 他手法参考比較

### 目的・示唆

> 既存手法は提案手法の代替にならない。目的関数が異なるため直接置換できない

速度ランキングや精度優劣が目的ではない。解いている問題が異なることを定量的・定性的に明示し、関連研究との差別化を図る。

### 比較構成

| 群 | 手法 | 問題定義 |
|----|------|----------|
| 同一目的（主比較） | Phase 1, 従来 Apriori-window | 密集区間・バスケット構造 |
| 異目的（参考） | Apriori, FP-Growth, Eclat, LCM | 全体支持度ベース |
| 異目的（参考） | PFPM, PPFPM, LPFIM, LPPM | 周期・ギャップベース |

### 実装独立性の担保（重要）

- 各手法は独立した実装を使用（SPMF / PAMI 等の公式実装を優先）
- 現在の共有コア（comparative_mining.rs）ではランタイム比較をしない
- 使用した実装名・バージョンを論文に明記する
- dev/test split で seed を分け、test 固定で評価する

### 評価構成

| 系統 | 指標 |
|------|------|
| 品質 | GTSPR, Observed Spurious（可能な範囲で） |
| 効率 | wall-clock, peak RSS（独立実装が確保できた手法のみ） |
| 運用性 | timeout 率, 失敗率 |

### 図表

- λ 曲線: 品質指標の挙動
- Pareto 図: 品質 vs 時間
- サマリ表: 各手法の平均・分散・timeout 率

### 解釈ルール

- 異目的群は「代替可能性の参考」として位置づける
- E4 単独で「提案法が常に最速/最良」とは主張しない
- 主結論は E0〜E3 から導き、E4 は補強として位置づける

## E6: 実データ検証（UCI Online Retail II）

### 目的・示唆

> 合成データで示した傾向が実購買データでも成立する。提案手法は実運用データに適用可能

外的妥当性の確認。合成データの設定（Poisson 生成・カテゴリ構造）が現実に対応していることを示す。代表パターンの定性分析で偽共起の実例を示す。

### データ概要

| 項目 | 内容 |
|------|------|
| データセット | UCI Online Retail II |
| 期間 | 2009-12〜2011-12 |
| 行数 | 約 100 万行 |
| バスケット定義 | InvoiceNo（1 回の購買イベント） |
| トランザクション定義 | 日付（同日の全購買を集約） |

### 前処理方針

1. キャンセル行（InvoiceNo が `C` 始まり）除外
2. Quantity ≤ 0 除外
3. InvoiceDate から日付抽出
4. 日付ごとにトランザクション ID 採番
5. InvoiceNo ごとにバスケット ID 採番
6. バスケット構造付き形式に変換（`item item | item ...`）

### 実験計画

#### E6-1: 従来法 vs Phase 1（主実験）

- 固定: min_support=50, window_size=90（日）
- 指標: Observed Spurious, 検出パターン数, 代表パターンの定性確認

#### E6-2: パラメータ感度

- sweep: min_support ∈ {10, 20, 50, 100, 200}, window_size ∈ {30, 90, 180}
- 指標: 検出パターン数, 密集区間数, 実行時間

#### E6-3: スケーラビリティ

- 期間切り取りで規模変更: 1 ヶ月 / 6 ヶ月 / 12 ヶ月 / 全期間
- 指標: 実行時間, peak RSS

### 期待される結論

- 実データでも従来法が多くの偽共起候補を検出し、提案法が抑制することを確認
- 代表パターンの定性分析で偽共起の実例を示す（例: 異なる顧客の購買が混入した組合せ）
